# 09. 状态、上下文与记忆

## 目标

本章处理 Agent 的记忆问题：

- 当前任务状态。
- 消息历史。
- 上下文裁剪。
- 简单长期记忆。

## 短期状态

短期状态可以先放内存：

```ts
export class InMemoryAgentState {
  private state: AgentState = {
    status: "idle",
    messages: [],
    steps: [],
  };

  get(): AgentState {
    return this.state;
  }
}
```

短期状态保存：

- 用户目标。
- 最近消息。
- 工具结果。
- 当前计划。
- 执行步骤。

## 上下文裁剪

消息太长会导致成本高、速度慢、效果差。

简单裁剪：

```ts
function trimMessages(messages: ChatMessage[], maxMessages = 20): ChatMessage[] {
  if (messages.length <= maxMessages) return messages;

  return [
    messages[0],
    {
      role: "system",
      content: "Earlier messages were omitted.",
    },
    ...messages.slice(-maxMessages + 1),
  ];
}
```

## 工具输出裁剪

工具输出必须限制：

- 文件内容截断。
- 搜索结果限制条数。
- 命令输出限制字符数。
- 大 JSON 做摘要。

## 简单长期记忆

先用 JSON 文件：

```ts
export type MemoryItem = {
  id: string;
  text: string;
  tags: string[];
  createdAt: string;
};
```

适合保存：

- 用户偏好。
- 项目测试命令。
- 常见错误处理方式。
- 历史任务摘要。

## 什么时候用向量数据库

当出现这些需求时再考虑：

- 记忆数量很多。
- 需要语义搜索。
- 需要跨项目复用经验。
- 简单关键词搜索不够用。

## 本章验收

你应该能：

- 保存任务状态。
- 控制上下文长度。
- 截断工具输出。
- 保存和读取简单长期记忆。
