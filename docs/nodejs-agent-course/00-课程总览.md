# 00. 课程总览

## 本课程要做什么

本课程会带你从零实现一个 Node.js 智能体。最终产物不是玩具提示词，而是一个具备基本工程结构的智能体运行时：

```text
CLI / TUI
  ↓
Agent Runtime
  ├─ LLM Client
  ├─ Tool Registry
  ├─ State Store
  ├─ Permission Manager
  └─ Logger
  ↓
Workspace / Shell / APIs / Memory
```

## 最终能力

学完后，你应该能实现：

- 一个命令行 Agent。
- 一个统一的 LLM 调用层。
- 一个可注册工具的工具系统。
- 文件读取、文件搜索、命令执行工具。
- Agent 的多轮工具调用循环。
- 基础权限控制。
- 上下文裁剪与简单记忆。
- 一个 mini coding agent。
- 可观察日志和基础测试策略。

## 不急着做什么

一开始不要急着做：

- 多 Agent 协作。
- 复杂 TUI。
- 向量数据库。
- 自动提交 PR。
- 完全自主修改系统环境。

这些都可以后面加。第一阶段最重要的是把 Agent 的闭环跑通。

## 学习节奏

建议节奏：

- 第 1 天：完成 01 到 04，跑通最小 CLI 和第一次真实模型调用。
- 第 2 天：完成 05 到 09，跑通客户端封装和工具调用循环。
- 第 3 天：完成 10 到 11，加入状态、记忆和工作流。
- 第 4 天：完成 12，做 mini coding agent。
- 第 5 天：完成 13，补日志、测试、TUI 和工程化。

## 验收标准

最终你应该能输入：

```text
帮我检查这个项目的测试为什么失败
```

Agent 能够：

1. 查看项目结构。
2. 读取配置文件。
3. 运行测试命令。
4. 分析失败输出。
5. 搜索相关代码。
6. 给出修复建议，或在确认后修改文件。
