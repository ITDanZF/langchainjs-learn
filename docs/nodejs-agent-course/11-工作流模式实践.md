# 10. 工作流模式实践

## 目标

本章学习三个比自主 Agent 更稳定的模式：

- 提示词链
- 路由
- 评估优化循环

## 提示词链

适合固定步骤任务。

示例：生成文档。

```text
用户需求
  ↓
生成大纲
  ↓
检查大纲
  ↓
生成正文
  ↓
润色
```

实现建议：

- 每一步单独 prompt。
- 每一步输出结构化。
- 中间结果可检查。

## 路由

适合输入类型清晰的任务。

```text
输入
  ↓
分类器
  ├─ 文档任务
  ├─ 代码任务
  └─ 问答任务
```

实现建议：

- 分类结果用 JSON schema。
- 每个路由有不同工具权限。
- 分类失败时走 fallback。

## 评估优化循环

适合能迭代改进的任务。

```text
生成答案
  ↓
评估答案
  ├─ 通过 → 输出
  └─ 不通过 → 带反馈重试
```

Coding agent 中常见版本：

```text
修改代码 → 运行测试 → 根据失败继续修改
```

## 为什么先学工作流

工作流的优点：

- 更稳定。
- 更容易测试。
- 更容易调试。
- 成本更可控。

自主 Agent 应该建立在这些模式之上，而不是替代它们。

## 本章验收

你应该能实现：

- 一个链式文档生成流程。
- 一个任务分类路由器。
- 一个带评估反馈的迭代流程。
