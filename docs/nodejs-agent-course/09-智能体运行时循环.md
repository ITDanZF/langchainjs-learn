# 08. 智能体运行时循环

## 目标

本章实现智能体最重要的闭环：

```text
LLM → tool_call → execute tool → tool_result → LLM → final answer
```

## 状态类型

```ts
export type AgentStatus =
  | "idle"
  | "running"
  | "waiting_approval"
  | "done"
  | "failed";

export type AgentState = {
  status: AgentStatus;
  messages: ChatMessage[];
  steps: AgentStep[];
};
```

## Runtime 结构

```ts
export type AgentRuntimeOptions = {
  llm: LLMClient;
  tools: ToolRegistry;
  logger: Logger;
  maxIterations?: number;
};
```

## 核心循环

```ts
for (let i = 0; i < maxIterations; i++) {
  const response = await llm.complete(messages);

  if (!response.toolCalls?.length) {
    return response.content;
  }

  for (const toolCall of response.toolCalls) {
    const output = await tools.execute(toolCall.name, toolCall.input);
    messages.push({
      role: "tool",
      content: JSON.stringify({ toolCall, output }),
    });
  }
}
```

## 必须有停止条件

停止条件包括：

- 模型给出最终回答。
- 达到最大迭代次数。
- 工具连续失败。
- 需要用户确认。
- 成本超过限制。

没有停止条件的 Agent 很容易陷入循环。

## 工具错误处理

工具失败不要直接崩溃。应该把错误作为 tool result 返回给模型：

```ts
messages.push({
  role: "tool",
  content: JSON.stringify({
    toolName,
    error: errorMessage,
  }),
});
```

这样模型可以根据错误调整下一步。

## 本章验收

你应该能：

- 让模型请求工具调用。
- 执行工具。
- 把工具结果放回上下文。
- 继续下一轮模型调用。
- 在最大次数后停止。
