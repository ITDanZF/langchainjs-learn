# 01. 智能体基础认知

## 智能体是什么

在本课程里，智能体指的是一个能围绕目标持续行动的系统：

```text
目标
  ↓
观察当前状态
  ↓
思考下一步
  ↓
调用工具或回答用户
  ↓
接收反馈
  ↓
继续循环
```

LLM 负责推理和决策，Node.js 程序负责工具执行、状态管理和安全边界。

## 工作流和智能体的区别

Workflow 是开发者预先写好的流程：

```text
输入 → 步骤 1 → 步骤 2 → 步骤 3 → 输出
```

Agent 是模型根据环境反馈动态决定路径：

```text
输入 → LLM 决策 → 工具结果 → LLM 再决策 → ...
```

实际项目中，不要迷信全自主。很多任务用 Workflow 更稳定。

## 智能体的四个核心构件

### 1. LLM

LLM 是控制中心，负责：

- 理解用户目标。
- 制定计划。
- 选择工具。
- 阅读工具结果。
- 生成最终回答。

### 2. Tools

工具让 Agent 能操作外部世界，例如：

- 读文件。
- 搜索代码。
- 执行测试。
- 查询 API。
- 写文件。

### 3. State

状态保存 Agent 正在做什么：

- 用户目标。
- 消息历史。
- 工具调用记录。
- 当前计划。
- 任务是否完成。

### 4. Feedback

反馈让 Agent 能修正行为：

- 文件内容。
- 命令输出。
- 测试失败。
- API 返回。
- 用户补充说明。

## 为什么 Node.js 适合做智能体

Node.js 适合做 Agent 的原因：

- 生态丰富，适合 CLI、TUI 和 Web。
- 调用 HTTP API 很方便。
- 操作文件系统和子进程方便。
- TypeScript 可以约束复杂结构。
- 很适合做开发者工具。

## 本课程的设计原则

1. 先实现简单闭环，再增加复杂能力。
2. 所有工具必须有 schema。
3. 所有高风险操作必须有权限控制。
4. 所有关键步骤必须能被日志看到。
5. Agent 必须有停止条件。
