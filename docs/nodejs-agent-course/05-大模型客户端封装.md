# 05. 大模型客户端封装

## 本章目标

上一章已经直接在 `src/index.ts` 里调用了真实 DeepSeek API：

```text
index.ts
  ↓
callDeepSeek(messages)
  ↓
fetch("https://api.deepseek.com/chat/completions")
  ↓
返回 assistant 文本
```

这说明 API Key、模型名、请求地址和响应解析已经跑通了。

本章要做的是把这段真实调用从 `index.ts` 中抽离出来，封装成统一的 LLM 客户端：

```text
index.ts
  ↓
llm.complete(messages)
  ↓
DeepSeekLLMClient
  ↓
fetch DeepSeek API
  ↓
返回 LLMResponse
```

封装之后，CLI 主流程不再关心底层模型供应商。以后如果要从 DeepSeek 换成 OpenAI、本地模型或 Mock 模型，只需要替换客户端实现。

## 1. 为什么现在适合封装

第 04 章已经让我们知道真实模型调用至少包含这些内容：

- 请求地址。
- 请求方法。
- 请求头。
- API Key。
- 模型名。
- `messages` 请求体。
- HTTP 错误处理。
- 响应 JSON 解析。
- `choices[0].message.content` 取值。

这些内容如果一直放在 `src/index.ts`，会让入口文件越来越重。

所以现在可以把它们抽到独立模块里。这个顺序更自然：

```text
先跑通真实调用
  ↓
再发现边界
  ↓
最后封装边界
```

## 2. 创建 llm 目录

如果还没有目录，先创建：

```bash
mkdir src/llm
```

本章会新增：

```text
src/
  llm/
    client.ts
```

## 3. 定义统一返回类型

创建 `src/llm/client.ts`：

```ts
import type { ChatMessage } from "../types";

export type LLMResponse = {
  content: string;
};
```

为什么不直接返回字符串？

因为后续模型返回的内容可能不只有文本，还可能有：

- 工具调用请求。
- token 用量。
- 模型名称。
- 原始响应。
- 错误信息。

所以先用对象包装：

```ts
{
  content: string
}
```

后面扩展会更方便。

## 4. 定义 LLMClient 接口

继续添加：

```ts
export type LLMClient = {
  complete(messages: ChatMessage[]): Promise<LLMResponse>;
};
```

这个接口表达的是：

```text
给我一组 messages
  ↓
我异步返回一个模型回复
```

`Promise` 是必须的，因为真实模型调用是网络请求。

## 5. 实现 DeepSeekLLMClient

把上一章 `callDeepSeek(messages)` 的逻辑移动到类里面：

```ts
import { config } from "../config";
import type { ChatMessage } from "../types";

export type LLMResponse = {
  content: string;
};

export type LLMClient = {
  complete(messages: ChatMessage[]): Promise<LLMResponse>;
};

export class DeepSeekLLMClient implements LLMClient {
  async complete(messages: ChatMessage[]): Promise<LLMResponse> {
    const response = await fetch("https://api.deepseek.com/chat/completions", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        Authorization: `Bearer ${config.DEEPSEEK_API_KEY}`,
      },
      body: JSON.stringify({
        model: config.DEEPSEEK_MODEL,
        messages,
      }),
    });

    if (!response.ok) {
      const errorText = await response.text();
      throw new Error(`DeepSeek 请求失败：${response.status} ${errorText}`);
    }

    const data = await response.json();
    const content = data.choices?.[0]?.message?.content ?? "";

    return { content };
  }
}
```

这段代码和上一章的区别是：

- 上一章返回 `string`。
- 这一章返回 `LLMResponse`。
- 上一章函数直接放在 `index.ts`。
- 这一章封装成 `DeepSeekLLMClient`。

## 6. 增加 MockLLMClient

真实模型适合验证最终效果，但开发时不一定每次都想消耗 API 调用。

所以可以再提供一个 Mock 客户端：

```ts
export class MockLLMClient implements LLMClient {
  async complete(messages: ChatMessage[]): Promise<LLMResponse> {
    const lastUserMessage = [...messages]
      .reverse()
      .find((message) => message.role === "user");

    return {
      content: lastUserMessage ? `收到：${lastUserMessage.content}` : "我没有收到用户输入。",
    };
  }
}
```

注意：Mock 不是这一章的主角。它只是一个备用实现，用来说明 `LLMClient` 接口的好处。

同一个接口下面可以有不同实现：

```text
LLMClient
  ├─ DeepSeekLLMClient
  └─ MockLLMClient
```

## 8. 改造 index.ts

上一章 `src/index.ts` 里有一个 `callDeepSeek(messages)` 函数。

现在可以删掉这个函数，改成导入客户端：

```ts
import { config } from "./config";
import { DeepSeekLLMClient } from "./llm/client";
import type { ChatMessage } from "./types";
```

创建客户端：

```ts
const llm = new DeepSeekLLMClient();
```

调用客户端：

```ts
const response = await llm.complete(messages);
```

然后把回复追加进消息列表：

```ts
addMsg({
  role: "assistant",
  content: response.content,
});
```

## 9. index.ts 完整代码

```ts
import { config } from "./config";
import { DeepSeekLLMClient } from "./llm/client";
import type { ChatMessage } from "./types";

const input = process.argv.slice(2).join(" ").trim();
const llm = new DeepSeekLLMClient();

function addMsg(msg: ChatMessage) {
  messages.push(msg);
}

// 缓存历史消息
const messages: ChatMessage[] = [
  {
    role: "system",
    content: "你是一个命令行agent!",
  },
];

if (!input) {
  console.error("请输入要执行的命令");
  process.exit(1);
}

if (!config.DEEPSEEK_API_KEY) {
  console.error("缺少 DEEPSEEK_API_KEY，请先在 .env 中配置。这个值不能提交到 Git。");
  process.exit(1);
}

addMsg({
  role: "user",
  content: input,
});

const response = await llm.complete(messages);

addMsg({
  role: "assistant",
  content: response.content,
});

console.log("当前消息列表：", messages);
```

现在 `index.ts` 不再关心 DeepSeek 的请求地址、请求头和响应格式。它只知道：

```ts
const response = await llm.complete(messages);
```

这就是封装的价值。

## 10. 如何切换到 Mock

如果开发时不想调用真实模型，可以把导入改成：

```ts
import { MockLLMClient } from "./llm/client";
```

再把创建客户端改成：

```ts
const llm = new MockLLMClient();
```

后面的主流程不用变。

这说明 `index.ts` 依赖的是“客户端能力”，不是某个具体模型。
