# 06. 工具系统设计

## 目标

本章实现一个最小工具系统，让 Agent 可以注册、发现和执行工具。

在当前项目里，工具系统由两个文件组成：

- `src/tools/types.ts`：定义工具的结构，也就是一个工具需要暴露哪些组件信息。
- `src/tools/Tools.ts`：维护工具注册表，负责注册、查找、列出和执行工具。

## 工具接口

创建 `src/tools/types.ts`：

```ts
import { z } from "zod";

export type ToolPermission = "read" | "write" | "execute_safe" | "execute_risky";

export type ToolDefinition<Input = unknown, Output = unknown> = {
  name: string;
  description: string;
  permission: ToolPermission;
  schema: z.ZodType<Input>;
  execute: (input: Input) => Promise<Output>;
};
```

一个工具必须包含：

- `name`：工具名称，模型调用工具时使用它。
- `description`：工具说明，告诉模型这个工具适合在什么时候使用。
- `permission`：权限等级，用来区分读、写、安全执行、风险执行。
- `schema`：参数校验规则，用 Zod 检查模型传入的参数是否可信。
- `execute`：真实执行逻辑，接收校验后的参数并返回执行结果。

## echo 工具的组件信息

工具注册表里保存的不是一个普通函数，而是一份完整的工具组件信息。本章只注册一个 `echo` 工具：

```ts
const tool: ToolDefinition = {
  name: "echo",
  description: "Return the text provided by the user.",
  permission: "read",
  schema: z.object({
    text: z.string(),
  }),
  async execute(input) {
    return {
      text: input.text,
    };
  },
};
```

这几个字段分别服务于不同阶段：

| 字段 | 给谁用 | 作用 |
| --- | --- | --- |
| `name` | 模型、工具注册表 | 唯一标识工具，执行时按名称查找 |
| `description` | 模型 | 帮助模型判断什么时候该调用这个工具 |
| `permission` | 权限系统 | 决定工具调用是否需要拦截、确认或拒绝 |
| `schema` | 执行器 | 在真正执行前校验参数 |
| `execute` | 工具系统 | 执行具体能力并返回结果 |

这也是为什么工具不能只注册一个函数。即使是最简单的 `echo`，Agent 也需要知道它叫什么、什么时候用、参数长什么样，以及真正执行时返回什么。

## Tool Registry

创建 `src/tools/Tools.ts`：

```ts
import { ToolDefinition } from "./types";

export default class Tools {
  private tools: Map<string, ToolDefinition> = new Map();

  registerTool(tool: ToolDefinition) {
    if (this.tools.has(tool.name)) {
      throw new Error(`Tool with name '${tool.name}' is already registered.`);
    }
    this.tools.set(tool.name, tool);
  }

  getTool(name: string): ToolDefinition {
    const tool = this.tools.get(name);
    if (!tool) {
      throw new Error(`Tool with name '${name}' is not registered.`);
    }
    return tool;
  }

  listTools(): ToolDefinition[] {
    return Array.from(this.tools.values());
  }

  async executeTool(name: string, input: unknown): Promise<unknown> {
    const tool = this.getTool(name);
    const parsed = tool.schema.parse(input);
    return tool.execute(parsed);
  }
}
```

这里的 `Map<string, ToolDefinition>` 就是工具注册表：

- key 是工具名，例如 `read_file`。
- value 是完整的工具组件信息，也就是 `ToolDefinition`。

执行工具时不要直接相信外部输入，而是先经过：

```ts
const parsed = tool.schema.parse(input);
```

只有通过 Zod 校验后的 `parsed` 才会传给工具的 `execute`。

## 注册工具示例

本章只注册一个 `echo` 工具。它接收一段文本，并原样返回这段文本：

```ts
import { z } from "zod";
import Tools from "./tools/Tools";

const tools = new Tools();

tools.registerTool({
  name: "echo",
  description: "Return the text provided by the user.",
  permission: "read",
  schema: z.object({
    text: z.string(),
  }),
  async execute(input) {
    return {
      text: input.text,
    };
  },
});
```

注册完成后，可以列出当前系统已经具备的工具能力：

```ts
const registeredTools = tools.listTools();
```

这份列表后面可以交给模型，让模型知道当前 Agent 有哪些可用组件。当前阶段里面只有一个工具：`echo`。

执行工具：

```ts
const result = await tools.executeTool("echo", {
  text: "hello agent",
});
```

返回结果：

```json
{
  "text": "hello agent"
}
```

这个例子已经完成了工具系统最小闭环：

```text
注册 echo 工具
  ↓
按名称找到 echo
  ↓
用 schema 校验输入参数
  ↓
执行 echo
  ↓
拿到工具返回结果
```

后续章节再继续把 `echo` 换成真正有用的文件读取、搜索和命令执行工具。

## 工具设计原则

1. 名称要短且明确。
2. 描述要告诉模型何时使用。
3. 参数尽量少。
4. 参数名要直观。
5. 返回值要结构化。
6. 错误要可读。
7. 输出要限制长度。

## 工具描述示例

不好：

```text
Read stuff.
```

较好：

```text
Read a UTF-8 text file from the current workspace. Use this when you need exact file contents.
```

## 本章验收

你应该能：

- 注册工具。
- 按名称查找工具。
- 列出已注册的 `echo` 工具信息。
- 校验参数。
- 调用 `echo` 并返回结构化结果。
