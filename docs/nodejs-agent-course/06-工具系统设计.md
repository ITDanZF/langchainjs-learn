# 05. 工具系统设计

## 目标

本章实现 Tool Registry，让 Agent 可以注册、发现和执行工具。

## 工具接口

创建 `src/tools/registry.ts`：

```ts
import { z } from "zod";

export type ToolPermission = "read" | "write" | "execute_safe" | "execute_risky";

export type ToolDefinition<Input = unknown, Output = unknown> = {
  name: string;
  description: string;
  permission: ToolPermission;
  schema: z.ZodType<Input>;
  execute(input: Input): Promise<Output>;
};
```

一个工具必须包含：

- `name`：模型调用时使用的名称。
- `description`：给模型看的说明。
- `permission`：权限等级。
- `schema`：参数校验。
- `execute`：真实执行逻辑。

## Tool Registry

```ts
export class ToolRegistry {
  private tools = new Map<string, ToolDefinition>();

  register(tool: ToolDefinition): void {
    if (this.tools.has(tool.name)) {
      throw new Error(`Tool already registered: ${tool.name}`);
    }
    this.tools.set(tool.name, tool);
  }

  get(name: string): ToolDefinition | undefined {
    return this.tools.get(name);
  }

  list(): ToolDefinition[] {
    return [...this.tools.values()];
  }

  async execute(name: string, input: unknown): Promise<unknown> {
    const tool = this.get(name);
    if (!tool) throw new Error(`Unknown tool: ${name}`);

    const parsed = tool.schema.parse(input);
    return tool.execute(parsed);
  }
}
```

## 工具设计原则

1. 名称要短且明确。
2. 描述要告诉模型何时使用。
3. 参数尽量少。
4. 参数名要直观。
5. 返回值要结构化。
6. 错误要可读。
7. 输出要限制长度。

## 工具描述示例

不好：

```text
Read stuff.
```

较好：

```text
Read a UTF-8 text file from the current workspace. Use this when you need exact file contents.
```

## 本章验收

你应该能：

- 注册工具。
- 按名称查找工具。
- 校验参数。
- 执行工具并返回结构化结果。
