# 12. 终端界面与工程化

## 目标

本章把 Agent 从脚本变成可长期使用的开发者工具。

## 为什么需要终端界面

Agent 运行时会发生很多事情：

- 模型正在思考。
- 工具正在调用。
- 命令正在执行。
- 需要用户确认。
- 出现错误。
- 计划发生变化。

终端界面可以让这些状态可见。

## 终端界面应该显示什么

建议显示：

- 对话消息。
- 当前计划。
- 工具调用列表。
- 工具结果摘要。
- 权限确认。
- 错误提示。
- token / cost / duration。

## 推荐技术

可选：

- `ink`：React 思维，适合现代 CLI。
- `blessed`：底层能力强，适合复杂终端布局。

建议先用普通 CLI，把 Runtime 做稳定后再做 TUI。

## 日志

至少记录：

- LLM 请求摘要。
- LLM 响应摘要。
- 工具调用。
- 工具结果。
- 错误。
- 状态变化。

日志不要记录 API Key。

## 测试

测试分三层：

1. 单元测试：工具、权限、路径限制。
2. 集成测试：LLM mock + 工具执行循环。
3. 任务评估：准备一组真实任务跑成功率。

## 配置

建议支持：

- 模型名称。
- 最大迭代次数。
- 工作区路径。
- 命令 allowlist。
- 是否自动执行安全命令。
- 日志级别。

## 生产清单

- [ ] 文件路径限制在 workspace 内。
- [ ] 所有工具有 schema。
- [ ] 工具输出有长度限制。
- [ ] 命令有 allowlist。
- [ ] 写操作需要确认。
- [ ] 删除操作强确认或禁用。
- [ ] 有最大迭代次数。
- [ ] 有取消任务能力。
- [ ] 有日志。
- [ ] 有测试。
- [ ] API Key 不进日志。
