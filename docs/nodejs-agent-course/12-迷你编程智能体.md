# 11. 迷你编程智能体

## 目标

本章把前面的模块组合成一个迷你编程智能体。

用户输入：

```text
帮我修复当前项目里失败的测试
```

Agent 需要能：

- 查看项目结构。
- 读取配置文件。
- 运行测试。
- 分析错误。
- 搜索代码。
- 修改文件。
- 再次运行测试。

## 基本流程

```text
1. list_files 查看结构
2. read_file 读取 package.json
3. run_command 运行测试
4. search_text 搜索错误关键词
5. read_file 读取相关文件
6. 生成修改计划
7. write_file 修改文件
8. run_command 再测
9. 输出总结
```

## 先不要自动写文件

第一版建议只做到：

- 搜索。
- 读取。
- 运行测试。
- 给出修改建议。

第二版再加 `write_file`，并且必须确认。

## write_file 工具设计

输入：

```ts
type WriteFileInput = {
  path: string;
  content: string;
};
```

要求：

- 只能写 workspace 内文件。
- 写之前展示 diff。
- 用户确认后再写。
- 记录写入历史。

## 测试反馈循环

核心是：

```text
修改 → 测试 → 失败信息 → 再修改
```

需要限制：

- 最大修复轮数。
- 连续失败次数。
- 命令超时。
- 修改文件数量。

## 输出总结

最终回答应该包含：

- 做了什么。
- 修改了哪些文件。
- 测试是否通过。
- 如果未通过，还剩什么问题。

## 本章验收

你应该能完成一个小任务：

```text
查找项目里的 TODO，并告诉我每个 TODO 在哪里
```

进阶任务：

```text
运行测试并解释失败原因
```

最终任务：

```text
修复一个简单失败测试
```
