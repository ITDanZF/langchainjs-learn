# 05b. 迭代器与异步生成器

## 本章目标

上一章把 `callDeepSeek` 封装成了 `DeepSeekLLMClient`，用回调函数 `onChunk` 实现流式输出。

本章学习一种更优雅的流式数据处理方式——**异步生成器（Async Generator）**，并尝试把 DeepSeek 的流式输出改造成生成器形式。

学完本章你会理解：

- 迭代器（Iterator）和可迭代对象（Iterable）
- 生成器函数（Generator）和 `yield`
- 异步生成器（Async Generator）和 `for await...of`
- 为什么 SSE 流式输出特别适合用异步生成器表达

---

## 1. 从数组说起：什么是可迭代

数组可以直接用 `for...of` 遍历：

```ts
const arr = ["你", "好", "，", "世", "界"];
for (const char of arr) {
  console.log(char);
}
```

底层原因是：数组实现了**可迭代协议（Iterable Protocol）**，它有一个特殊方法 `[Symbol.iterator]()`，返回一个**迭代器（Iterator）**。

迭代器每次调用 `next()`，返回这样的对象：

```ts
{
  value: "你",  // 当前值
  done: false   // 是否遍历结束
}
```

当 `done` 为 `true` 时，循环结束。

`for...of` 本质上就是帮你不断调用 `next()`。

---

## 2. 生成器：简化迭代器编写

手动实现迭代器比较繁琐。ES6 引入**生成器函数（Generator Function）**，用 `function*` 声明，用 `yield` 产出值：

```ts
function* countTo(n: number) {
  for (let i = 1; i <= n; i++) {
    yield i;
  }
}

for (const num of countTo(5)) {
  console.log(num); // 1, 2, 3, 4, 5
}
```

`yield` 有两个神奇效果：

1. **产出值**：把值交给 `for...of` 的循环变量。
2. **暂停执行**：函数停在 `yield` 处，等外部消费完这个值后，再回来继续执行。

你可以把生成器想象成一个"可以暂停的函数"。

---

## 3. 异步生成器：处理流式数据

如果数据不是立刻就有，而是异步到达的（比如网络流、数据库查询），就用**异步生成器**：

```ts
async function* delayedNumbers(max: number) {
  for (let i = 1; i <= max; i++) {
    await new Promise((resolve) => setTimeout(resolve, 200));
    yield i;
  }
}
```

消费异步生成器要用 `for await...of`：

```ts
for await (const num of delayedNumbers(5)) {
  console.log(num);
}
```

这里的 `await` 会自动等待每次 `yield` 返回的 Promise，代码写起来和同步遍历几乎一样。

---

## 4. 为什么 SSE 适合用异步生成器

DeepSeek 的流式接口返回的是 SSE（Server-Sent Events）数据：

```text
data: {"choices":[{"delta":{"content":"你"}}]}
data: {"choices":[{"delta":{"content":"好"}}]}
data: [DONE]
```

数据是一段一段到达的。用回调写是这样：

```ts
await DeepSeek(messages, (chunk) => {
  process.stdout.write(chunk);
});
```

用异步生成器写是这样：

```ts
for await (const chunk of DeepSeekStream(messages)) {
  process.stdout.write(chunk);
}
```

区别：

- **回调版**：控制权在 `DeepSeek` 函数内部，它决定什么时候调用你的回调。
- **生成器版**：控制权在调用方，你可以自己决定怎么消费每个 chunk，比如缓冲、打字机效果、取消、统计字数等。

生成器让"数据生产"和"数据消费"解耦，代码更清晰，也更容易测试。

### 5.1 练习 1：同步生成器

```ts
function* countTo(n: number) {
  for (let i = 1; i <= n; i++) {
    yield i;
  }
}

for (const num of countTo(5)) {
  console.log(num);
}
```

重点感受 `yield` 的"暂停"效果。

### 5.2 练习 2：异步生成器

```ts
async function* delayedNumbers(max: number) {
  for (let i = 1; i <= max; i++) {
    await new Promise((resolve) => setTimeout(resolve, 200));
    yield i;
  }
}

for await (const num of delayedNumbers(5)) {
  console.log(num);
}
```

注意这里是 `for await...of`，不是 `for...of`。

### 5.3 练习 3：模拟 SSE 流式输出

```ts
async function* fakeSSEStream(text: string) {
  const chars = text.split("");
  for (const char of chars) {
    await new Promise((resolve) => setTimeout(resolve, 50));
    yield char;
  }
}

let fullContent = "";
for await (const chunk of fakeSSEStream("你好，我是 AI 助手。")) {
  process.stdout.write(chunk);
  fullContent += chunk;
}
process.stdout.write("\n");
```

这就像 DeepSeek 的 SSE 流：每次 `yield` 一个字，调用方逐字打印。

### 5.4 练习 4：缓冲输出

```ts
let buffer = "";
const flush = () => {
  if (buffer) {
    process.stdout.write(buffer);
    buffer = "";
  }
};

for await (const chunk of fakeSSEStream(text)) {
  buffer += chunk;
  if (buffer.length >= 5) {
    flush();
  }
}
flush(); // 最后一定要刷新！
process.stdout.write("\n");
```

缓冲可以减少 I/O 次数，但记得循环结束后调用一次 `flush()`。

## 6. 把 DeepSeek 改造成生成器

理解了生成器后，可以把 `service/index.ts` 改造成生成器版本。

### 6.1 新增 `DeepSeekStream`

```ts
import { BaseMessage } from "../message/type";
import { config } from "../config";

export async function* DeepSeekStream(
  messages: BaseMessage[],
  signal?: AbortSignal,
): AsyncGenerator<string, string> {
  const response = await fetch("https://api.deepseek.com/chat/completions", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${config.DEEPSEEK_API_KEY}`,
    },
    body: JSON.stringify({
      model: config.DEEPSEEK_MODEL,
      messages,
      stream: true,
    }),
    signal,
  });

  if (!response.ok) {
    const errorText = await response.text();
    throw new Error(`DeepSeek 请求失败：${response.status} ${errorText}`);
  }

  const reader = response.body?.getReader();
  if (!reader) {
    throw new Error("Failed to get response reader");
  }

  let fullContent = "";
  const decoder = new TextDecoder("utf-8");

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value, { stream: true });
    for (const line of chunk.split("\n")) {
      const trimmed = line.trim();
      if (!trimmed || !trimmed.startsWith("data:")) continue;

      const jsonStr = trimmed.slice(5).trim();
      if (jsonStr === "[DONE]") break;

      try {
        const data = JSON.parse(jsonStr);
        const content = data.choices?.[0]?.delta?.content;
        if (content) {
          fullContent += content;
          yield content;
        }
      } catch (error) {
        if (jsonStr !== "") {
          console.error("SSE 解析失败:", jsonStr, error);
        }
      }
    }
  }

  return fullContent;
}
```

关键变化：

- 函数声明改成 `async function*`
- 返回类型改成 `AsyncGenerator<string, string>`
- 每次拿到 content 时用 `yield content` 交给调用方
- 循环结束时 `return fullContent`

### 6.2 调用方写法

```ts
import { DeepSeekStream } from "./service/index";

let aiMsg = "";
for await (const chunk of DeepSeekStream(newMsg)) {
  process.stdout.write(chunk);
  aiMsg += chunk;
}
process.stdout.write("\n");

AIMessage(aiMsg);
```

### 6.3 保留回调兼容层

如果你还想保留原来的回调接口，可以包一层：

```ts
export async function DeepSeek(
  messages: BaseMessage[],
  onChunk?: (chunk: string) => void,
  signal?: AbortSignal,
): Promise<string> {
  let fullContent = "";
  for await (const chunk of DeepSeekStream(messages, signal)) {
    onChunk?.(chunk);
    fullContent += chunk;
  }
  return fullContent;
}
```

这样旧的调用代码不用改，但核心实现已经变成生成器了。

## 7. 常见误区

### 误区 1：`for await...of` 和 `for...of` 混用

异步生成器必须用 `for await...of`，否则会报错或拿不到值。

### 误区 2：在 `onChunk` 回调里 `await`

如果你保留回调版本，注意 `onChunk` 是同步回调：

```ts
// ❌ 不要这样写
await DeepSeek(newMsg, async (chunk) => {
  await sleep(10);
  process.stdout.write(chunk);
});
```

这样会导致流读取被阻塞。想做打字机效果，建议用生成器版本，在 `for await...of` 循环里控制节奏。

### 误区 3：忘记 `return`

生成器里的 `return` 值不会出现在 `for await...of` 循环中，需要手动收集：

```ts
let fullContent = "";
for await (const chunk of DeepSeekStream(messages)) {
  fullContent += chunk;
}
```

---

## 8. 小结

- **迭代器**：实现 `next()` 的对象，用于逐个取值。
- **可迭代对象**：实现 `[Symbol.iterator]()` 的对象，可以用 `for...of` 遍历。
- **生成器**：用 `function*` 和 `yield` 简化迭代器编写。
- **异步生成器**：用 `async function*` 处理异步数据流，用 `for await...of` 消费。
- **SSE 流式输出**：本质就是异步数据流，非常适合用异步生成器表达。

生成器让"数据怎么来"和"数据怎么用"解耦，是处理流式 AI 输出的利器。
