# 07. 记忆、检查点与多轮会话

## 本章目标

上一章的 LangGraph 可以处理一次请求，但复杂应用通常需要多轮会话。

本章会学习：

- 什么是 checkpointer。
- 什么是 `thread_id`。
- 如何让 LangGraph 保存会话状态。
- 如何在多轮对话中复用上下文。

## 1. 第一版的记忆问题

第一版课程里，我们自己维护：

```ts
const messages: ChatMessage[] = [];
```

每次用户输入，都把消息 push 进去。

这种方式可以理解原理，但真实应用会遇到问题：

- 进程重启后记忆丢失。
- 多个用户会话混在一起。
- 无法恢复某个会话。
- 难以观测每一步状态。

LangGraph 用 checkpointer 解决这些问题。

## 2. 使用 MemorySaver

修改 `src/agents/graph-agent.ts`：

```ts
import { MemorySaver } from "@langchain/langgraph";

const checkpointer = new MemorySaver();

export const graphAgent = new StateGraph(AgentState)
  .addNode("model", callModel)
  .addEdge("__start__", "model")
  .addEdge("model", "__end__")
  .compile({ checkpointer });
```

`MemorySaver` 是内存版检查点，适合学习和本地开发。

生产环境通常会换成数据库或持久化存储。

## 3. thread_id 是什么

调用图时传入：

```ts
const config = {
  configurable: {
    thread_id: "demo-user-1",
  },
};
```

`thread_id` 用来区分不同会话。

同一个 `thread_id` 会继续之前的上下文，不同 `thread_id` 会拥有不同记忆。

## 4. 多轮调用示例

```ts
import { HumanMessage } from "@langchain/core/messages";
import { graphAgent } from "./agents/graph-agent";

const config = {
  configurable: {
    thread_id: "demo-user-1",
  },
};

await graphAgent.invoke(
  {
    messages: [new HumanMessage("我叫小明")],
  },
  config,
);

const result = await graphAgent.invoke(
  {
    messages: [new HumanMessage("我叫什么？")],
  },
  config,
);

console.log(result.messages.at(-1)?.content);
```

如果记忆生效，模型应该能根据上下文回答“小明”。

## 5. 不同 thread_id 的隔离

```ts
const userA = {
  configurable: { thread_id: "user-a" },
};

const userB = {
  configurable: { thread_id: "user-b" },
};
```

`user-a` 和 `user-b` 的消息历史会分开保存。

这是构建真实多用户应用的基础。

## 6. 记忆不是无限上下文

有了 checkpointer，不代表可以无限塞消息。

真实应用还需要：

- 上下文裁剪。
- 摘要记忆。
- 长期存储。
- 检索式记忆。
- 隐私和权限控制。

本章先掌握会话状态保存，后面再结合 RAG 和工程化扩展。
