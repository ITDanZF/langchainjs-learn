# 07. 会话记忆：升级默认交互会话

## 本章目标

第 03 章已经实现了默认交互会话，但到目前为止它只是“连续输入”。每一轮之间还没有稳定的会话状态。

本章把已有交互会话升级为带 thread 和记忆的多轮会话：

```bash
npm run dev
npm run dev -- --thread dev
```

你会新增或修改：

```text
src/graph/task-graph.ts
src/memory/thread.ts
src/main.ts
```

## 1. 为什么需要 thread

企业 Agent 通常会同时服务多个任务、用户或项目。如果只有一份全局记忆，就会互相污染。

所以多轮会话必须引入线程：

```text
thread: dev     → 开发任务上下文
thread: docs    → 文档任务上下文
thread: support → 工单任务上下文
```

同一个 `thread_id` 内保留上下文，不同 `thread_id` 之间隔离。

## 2. 给 Graph 加 checkpointer

修改 `src/graph/task-graph.ts`，增加内存检查点：

```ts
import { MemorySaver } from "@langchain/langgraph";

const checkpointer = new MemorySaver();
```

然后在 `compile()` 时传入：

```ts
.compile({ checkpointer });
```

教学项目用 `MemorySaver` 足够。生产项目可以替换为 SQLite、Postgres、Redis 或 LangGraph 平台提供的持久化能力。

## 3. 支持 thread_id 调用

在 `src/graph/task-graph.ts` 中增加：

```ts
export async function runGraphMessage(input: string, threadId: string) {
  const graph = createTaskGraph();

  const result = await graph.invoke(
    { messages: [new HumanMessage(input)] },
    { configurable: { thread_id: threadId } },
  );

  return result.messages.at(-1)?.content ?? "";
}
```

`thread_id` 是会话记忆的关键。相同 `thread_id` 会自动接续历史状态。

## 4. 线程 ID 工具

创建 `src/memory/thread.ts`：

```ts
export function createThreadId(prefix = "thread") {
  return `${prefix}-${Date.now()}`;
}
```

后续可以扩展为：用户 ID、项目 ID、任务 ID 的组合。

## 5. 升级默认交互入口

修改 `src/main.ts`：

```ts
import { createInterface } from "node:readline/promises";
import { stdin as input, stdout as output } from "node:process";
import { Command } from "commander";
import { runGraphMessage } from "./graph/task-graph.js";
import { createThreadId } from "./memory/thread.js";

async function startThreadChatLoop(initialThreadId: string) {
  const rl = createInterface({ input, output });
  let threadId = initialThreadId;

  console.log(`进入会话：${threadId}`);
  console.log("输入 /exit 退出，/clear 开启新会话，/thread <id> 切换会话。\n");

  try {
    while (true) {
      const line = (await rl.question("> ")).trim();

      if (!line) continue;
      if (line === "/exit") break;

      if (line === "/clear") {
        threadId = createThreadId();
        console.log(`已切换到新会话：${threadId}`);
        continue;
      }

      if (line.startsWith("/thread ")) {
        threadId = line.slice("/thread ".length).trim();
        console.log(`已切换到会话：${threadId}`);
        continue;
      }

      const response = await runGraphMessage(line, threadId);
      console.log(response);
      console.log();
    }
  } finally {
    rl.close();
  }
}

async function main() {
  const program = new Command();

  program.name("chatAgent");
  program.version("0.1.0");
  program.option("-t, --thread <id>", "thread id", "default");

  program.action(async (options: { thread: string }) => {
    await startThreadChatLoop(options.thread);
  });

  await program.parseAsync(process.argv);
}

main();
```

注意：`--thread` 是传给你的程序，不是传给 npm，所以本地运行时需要写在 `--` 后面。

## 6. 验收

默认进入会话：

```bash
npm run dev
```

或者指定 thread：

```bash
npm run dev -- --thread demo
```

输入：

```text
> 列出当前项目文件
> 刚才 package.json 里有哪些脚本？
```

在指定 thread 的会话中，第二轮应该能理解“刚才”的上下文。

## 7. 企业级思考

记忆不是越多越好。企业 Agent 需要区分：

| 记忆类型 | 示例 | 生命周期 |
| --- | --- | --- |
| 短期上下文 | 当前对话消息 | 一个 thread |
| 任务状态 | 已执行步骤、工具结果 | 一个任务 |
| 用户偏好 | 输出语言、格式偏好 | 长期 |
| 企业知识 | 文档、制度、FAQ | 由 RAG 管理 |

下一章会实现企业知识的第一版：本地 Markdown RAG。
