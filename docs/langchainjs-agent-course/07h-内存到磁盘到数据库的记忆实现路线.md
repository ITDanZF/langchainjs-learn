# 07h. 内存到磁盘到数据库：短期记忆的三阶段实现路线

> 本章专门对应当前项目的实践路线：先做进程内内存记忆，让对话具备上下文；再做磁盘缓存，让 CLI 重启后能恢复；最后再接数据库和长期记忆，支持跨会话、跨设备、跨服务实例的存储。

## 1. 先确定路线

当前不要一上来就做数据库，也不要先做磁盘缓存。推荐顺序是：

```text
阶段 1：进程内内存记忆
  目标：同一次 CLI 运行中，模型能记住前几轮对话
  存储：Node.js 进程内 Map / 数组
  不写磁盘，不接数据库

阶段 2：磁盘 session 缓存
  目标：CLI 退出再启动后，同一个 thread 能恢复历史
  存储：~/.mini-agent/sessions/<thread-id>.jsonl
  仍然不接数据库

阶段 3：数据库与跨对话存储
  目标：跨机器、跨进程、跨会话持久化
  短期状态：PostgresSaver / SQLiteSaver 等 checkpointer
  长期记忆：PostgresStore / InMemoryStore / RedisStore 等 store
```

这条路线和官方文档并不冲突。官方 `MemorySaver` 本身也是内存 checkpointer，只是它服务于 `createAgent` / LangGraph；当前项目还没完全切到官方 agent，所以第一阶段先手写 `Memory` 更适合学习底层机制。

## 2. 官方概念对应关系

| 当前阶段 | 当前项目实现 | 官方概念 | 说明 |
| --- | --- | --- | --- |
| 阶段 1 | `Memory` class + `Map<string, BaseMessage[]>` | short-term memory | 只在当前进程内保存 |
| 阶段 2 | JSONL session 文件 | checkpoint 的本地简化版 | 让 thread 历史落盘 |
| 阶段 3A | `PostgresSaver` / `SQLiteSaver` | checkpointer | 保存 thread-level state |
| 阶段 3B | `PostgresStore` / `InMemoryStore` | store | 保存跨 thread 的长期记忆 |

一句话：

```text
checkpointer 管当前 thread 的状态；
store 管跨 thread 仍然有价值的长期信息。
```

## 3. 阶段 1：只做进程内内存记忆

目标非常小：

```text
同一次 CLI 运行中：
> 我叫张三
> 我刚才说我叫什么？

Agent 能回答“张三”。
```

此阶段不要做：

- 不写 JSON 文件。
- 不接 SQLite。
- 不接 PostgreSQL。
- 不做长期记忆。
- 不做语义搜索。
- 不做自动摘要。

### 3.1 Memory 数据结构

第一版可以先只支持一个默认 thread：

```ts
import { AIMessage, BaseMessage, HumanMessage } from "@langchain/core/messages";

export class Memory {
  private messages: BaseMessage[] = [];

  addUserMessage(content: string): void {
    this.messages.push(new HumanMessage(content));
  }

  addAiMessage(content: string): void {
    this.messages.push(new AIMessage(content));
  }

  getRecentMessages(limit = 20): BaseMessage[] {
    return this.messages.slice(-limit);
  }

  clear(): void {
    this.messages = [];
  }
}
```

如果想一步到位支持多个 thread，可以用：

```ts
private threads = new Map<string, BaseMessage[]>();
```

但第一版不必急。先让默认会话记住上下文。

### 3.2 Prompt 接入 history

`Model.ts` 需要把历史消息插入 prompt：

```ts
import { ChatPromptTemplate, MessagesPlaceholder } from "@langchain/core/prompts";

const prompt = ChatPromptTemplate.fromMessages([
  ["system", "你是一个 CLI 编程助手。"],
  new MessagesPlaceholder("history"),
  ["human", "{input}"],
]);
```

模型调用变成：

```ts
await chain.stream({
  input,
  history: memory.getRecentMessages(20),
});
```

注意：本轮用户输入只放进 `{input}`，不要同时放进 `history`。

### 3.3 主流程写回记忆

推荐流程：

```text
1. 从 memory 读取历史
2. 调用 model.stream({ input, history })
3. PrintStream 一边打印一边收集完整 assistant 文本
4. 本轮结束后写入 user message
5. 本轮结束后写入 ai message
```

伪代码：

```ts
const history = memory.getRecentMessages(20);

const stream = await model.stream({
  input,
  history,
});

const answer = await PrintStream(stream);

memory.addUserMessage(input);
memory.addAiMessage(answer);
```

为什么不是先把 user 写入 memory？

因为如果 prompt 已经有 `["human", "{input}"]`，再把当前输入放进 `history`，模型会看到两遍同样的问题。

### 3.4 阶段 1 验收

验收 1：默认会话能记住名字。

```text
> 我叫张三
> 我刚才说我叫什么？
```

预期：能回答“张三”。

验收 2：清空后不再记得。

```text
> /clear
> 我刚才说我叫什么？
```

预期：不能稳定回答“张三”。

验收 3：重启后丢失。

```text
退出 CLI
重新启动 CLI
> 我刚才说我叫什么？
```

预期：不记得。这是阶段 1 的正确行为，因为还没有磁盘缓存。

## 4. 阶段 2：再做磁盘 session 缓存

阶段 2 的目标是：

```text
CLI 重启后，同一个 thread 能恢复历史。
```

推荐文件：

```text
~/.mini-agent/sessions/default.jsonl
~/.mini-agent/sessions/<thread-id>.jsonl
```

每行一条消息：

```json
{"role":"human","content":"我叫张三","createdAt":"2026-07-02T00:00:00.000Z"}
{"role":"ai","content":"好的，我记住了。","createdAt":"2026-07-02T00:00:01.000Z"}
```

### 4.1 不直接序列化 BaseMessage

不要把 LangChain 的 `BaseMessage` 实例直接写进文件。建议使用稳定 DTO：

```ts
type PersistedMessage = {
  role: "human" | "ai" | "system" | "tool";
  content: string;
  createdAt: string;
  metadata?: Record<string, unknown>;
};
```

这样以后迁移到数据库时，数据结构更稳定。

### 4.2 阶段 2 验收

```text
> 我叫张三
退出 CLI
重新启动 CLI
> 我刚才说我叫什么？
```

预期：能回答“张三”。

此时仍然没有数据库。磁盘缓存只解决本机恢复问题。

## 5. 阶段 3：最后再接数据库

数据库阶段要拆成两条线。

### 5.1 数据库 checkpointer：保存短期 thread state

官方生产级短期记忆使用数据库 checkpointer：

```ts
import { PostgresSaver } from "@langchain/langgraph-checkpoint-postgres";

const checkpointer = PostgresSaver.fromConnString(process.env.POSTGRES_URI!);
await checkpointer.setup();
```

它解决的是：

```text
同一个 thread 的 graph state 如何跨进程、跨服务实例恢复？
```

对应当前项目里的：

```text
sessionId / threadId -> messages / state
```

### 5.2 数据库 store：保存跨对话长期记忆

跨对话长期记忆使用 store：

```ts
import { PostgresStore } from "@langchain/langgraph-checkpoint-postgres/store";

const store = PostgresStore.fromConnString(process.env.POSTGRES_URI!);
await store.setup();
```

它解决的是：

```text
新 thread 也应该知道什么？
```

例如：

```ts
await store.put(["users", userId, "preferences"], "language", {
  value: "中文",
});
```

### 5.3 阶段 3 不要一开始就自动记忆

数据库能保存，不代表应该保存一切。

长期记忆第一版建议做手动命令：

```text
/memory remember language 中文
/memory recall language
/memory forget language
/memory list
```

等权限、隐私、删除策略明确后，再考虑自动提取长期记忆。

## 6. 当前项目的下一步

现在最应该做的是阶段 1：

```text
1. 实现 src/Memory/index.ts 的进程内 messages
2. 修改 Model.ts，让 prompt 支持 history
3. 修改 PrintStream，让它返回完整 assistant 文本
4. 修改 main.ts，把每轮 user / ai 写回 Memory
5. 验证“我叫张三 / 我刚才说我叫什么”
```

等阶段 1 稳定后，再进入阶段 2 的 JSONL 文件缓存。
