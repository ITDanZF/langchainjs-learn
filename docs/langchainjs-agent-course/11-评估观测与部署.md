# 11. 评估、观测与部署

## 本章目标

能跑通 LangChain.js 应用只是第一步。真实项目还需要回答：

- 它为什么这样回答？
- 哪一步调用了哪个工具？
- 成本是多少？
- 改 Prompt 后质量有没有退化？
- 出错时如何定位？
- 应该部署成 CLI、API 还是后台任务？

本章会介绍评估、观测和部署的基础做法。

## 1. 为什么需要观测

Agent 应用不是普通函数调用。

一次用户请求中可能发生：

```text
模型调用
  ↓
工具调用
  ↓
模型调用
  ↓
检索
  ↓
模型调用
```

如果没有观测，很难知道错误发生在哪里。

## 2. LangSmith tracing

LangSmith 是 LangChain 生态中的观测和评估平台。

可以在 `.env` 中配置：

```env
LANGSMITH_TRACING=true
LANGSMITH_API_KEY=你的 LangSmith Key
LANGSMITH_PROJECT=langchain-agent-demo
```

配置后，LangChain.js 的运行链路可以被记录下来，包括：

- chain 输入输出。
- prompt 内容。
- model 调用。
- tool 调用。
- token 使用。
- 错误信息。

不要上传敏感数据到外部观测平台，生产环境要先确认数据合规要求。

## 3. 本地评估用例

即使不用 LangSmith，也应该维护一组本地评估用例。

创建 `src/evals/cases.ts`：

```ts
export const evalCases = [
  {
    input: "请总结 docs 目录下有哪些课程",
    expectedKeywords: ["Node.js", "智能体", "课程"],
  },
  {
    input: "查询 open 状态工单",
    expectedKeywords: ["T-1001", "open"],
  },
];
```

## 4. 简单评估脚本

创建 `src/evals/run-eval.ts`：

```ts
import { HumanMessage } from "@langchain/core/messages";
import { businessAgent } from "../agents/business-agent";
import { evalCases } from "./cases";

for (const item of evalCases) {
  const result = await businessAgent.invoke({
    messages: [new HumanMessage(item.input)],
  });

  const output = String(result.messages.at(-1)?.content ?? "");
  const passed = item.expectedKeywords.every((keyword) => output.includes(keyword));

  console.log({
    input: item.input,
    passed,
    output,
  });
}
```

这不是完美评估，但足够作为入门回归检查。

## 5. 成本控制

复杂 agent 成本可能来自：

- 多次模型调用。
- 长上下文。
- RAG 检索后塞入过多文档。
- 重复工具调用。
- 批量任务没有限流。

常见优化方式：

- 限制最大迭代次数。
- 控制检索片段数量。
- 对长文档先摘要再回答。
- 对重复问题做缓存。
- 在工具前加入路由判断。

## 6. 部署形态

LangChain.js 应用可以部署成多种形态：

- CLI：适合开发工具、内部脚本。
- HTTP API：适合接入 Web 前端或业务系统。
- Worker：适合后台批处理任务。
- Bot：适合企业微信、飞书、Slack。
- TUI：适合终端生产力工具。

课程里的 CLI 示例只是入口之一。

核心能力应该封装在 agent、chain、tool 模块里，方便换入口。

## 7. 生产化检查清单

上线前至少检查：

- API Key 不在代码仓库中。
- 工具权限有边界。
- 命令执行类工具需要审批。
- 日志不输出敏感数据。
- 典型任务有评估用例。
- 模型错误有重试或降级策略。
- RAG 文档更新流程明确。
- 成本和调用次数可观测。

## 8. 本章验收

完成后，你应该能：

- 理解为什么 agent 应用需要观测。
- 配置 LangSmith tracing。
- 编写简单本地评估用例。
- 理解 agent 成本来源。
- 选择合适的部署形态。
- 制定生产化检查清单。
