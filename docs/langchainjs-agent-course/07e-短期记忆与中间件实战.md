# 07e. 短期记忆与中间件实战：从 MemorySaver 到 trim / summary

> 本章聚焦短期记忆：如何让同一个 thread 里的对话连续，如何管理越来越长的消息历史。

## 1. 短期记忆的最小闭环

官方最小闭环是：

```ts
import { createAgent } from "langchain";
import { MemorySaver } from "@langchain/langgraph";

const agent = createAgent({
  model: "openai:gpt-5-mini",
  tools: [],
  checkpointer: new MemorySaver(),
});

const config = { configurable: { thread_id: "default" } };

await agent.invoke(
  { messages: [{ role: "user", content: "我叫张三" }] },
  config,
);

const result = await agent.invoke(
  { messages: [{ role: "user", content: "我叫什么？" }] },
  config,
);
```

第二次只传当前消息，但 agent 仍然能读到同一个 `thread_id` 下的历史 state。

## 2. `MemorySaver` 到底保存在哪里

`MemorySaver` 保存到当前 Node.js 进程的内存中。

这意味着：

- 不写磁盘。
- 不写数据库。
- 服务重启后丢失。
- 多进程之间不共享。

因此它适合学习，但不是生产持久化方案。

## 3. 短期记忆保存的不只是聊天

在官方 agent / LangGraph 中，短期记忆保存的是 thread state。

`messages` 是其中最常见的一部分：

```text
state = {
  messages: [...],
  可能还有 userId,
  可能还有当前任务状态,
  可能还有工具中间结果,
  可能还有 summary
}
```

这也是为什么官方说短期记忆会在 agent 被调用时、或者工具调用等 step 完成时更新。

## 4. 手写短期记忆时怎么贴近官方语义

当前项目还没有完整 `createAgent`，所以第一版可以手写：

```ts
import { AIMessage, BaseMessage, HumanMessage } from "@langchain/core/messages";

export class Memory {
  private threads = new Map<string, BaseMessage[]>();

  getMessages(threadId: string): BaseMessage[] {
    return [...(this.threads.get(threadId) ?? [])];
  }

  addUserMessage(threadId: string, content: string): void {
    this.append(threadId, new HumanMessage(content));
  }

  addAiMessage(threadId: string, content: string): void {
    this.append(threadId, new AIMessage(content));
  }

  getRecentMessages(threadId: string, limit = 20): BaseMessage[] {
    return this.getMessages(threadId).slice(-limit);
  }

  clear(threadId: string): void {
    this.threads.delete(threadId);
  }

  private append(threadId: string, message: BaseMessage): void {
    const messages = this.threads.get(threadId) ?? [];
    this.threads.set(threadId, [...messages, message]);
  }
}
```

这不是最终生产方案，但它让你先理解：

```text
thread_id -> messages[]
```

## 5. Prompt 如何消费历史

如果继续使用 `ChatPromptTemplate`，需要 `MessagesPlaceholder`：

```ts
import { ChatPromptTemplate, MessagesPlaceholder } from "@langchain/core/prompts";

const prompt = ChatPromptTemplate.fromMessages([
  ["system", "你是一个 CLI 编程助手。"],
  new MessagesPlaceholder("history"),
  ["human", "{input}"],
]);
```

调用：

```ts
const result = await chain.stream({
  input,
  history: memory.getRecentMessages(threadId, 20),
});
```

本轮用户输入只放在 `{input}`，不要提前写进 `history`。

## 6. 为什么需要裁剪

短期记忆会越来越长：

```text
第 1 轮 user / ai
第 2 轮 user / ai
第 3 轮 user / ai
...
```

问题：

- 上下文窗口有限。
- 请求越来越慢。
- 成本越来越高。
- 很早以前的无关内容会干扰模型。

最简单策略：

```ts
memory.getRecentMessages(threadId, 20);
```

这适合第一版。

## 7. 官方裁剪：`trimMessages`

官方推荐在模型调用前用 middleware 裁剪：

```ts
import { createAgent, createMiddleware, trimMessages } from "langchain";
import { RemoveMessage } from "@langchain/core/messages";
import { MemorySaver, REMOVE_ALL_MESSAGES } from "@langchain/langgraph";

const trimMessageHistory = createMiddleware({
  name: "TrimMessages",
  beforeModel: async (state) => {
    const trimmed = await trimMessages(state.messages, {
      maxTokens: 384,
      strategy: "last",
      startOn: "human",
      endOn: ["human", "tool"],
      tokenCounter: (messages) => messages.length,
    });

    return {
      messages: [new RemoveMessage({ id: REMOVE_ALL_MESSAGES }), ...trimmed],
    };
  },
});

const agent = createAgent({
  model: "openai:gpt-5-mini",
  tools: [],
  middleware: [trimMessageHistory],
  checkpointer: new MemorySaver(),
});
```

关键点：

- 裁剪发生在 `beforeModel`。
- 用 `RemoveMessage({ id: REMOVE_ALL_MESSAGES })` 先清空旧 messages。
- 再把裁剪后的 messages 写回 state。

## 8. 删除消息：`RemoveMessage`

如果想永久删除 state 中的消息，可以返回：

```ts
import { RemoveMessage } from "@langchain/core/messages";

return {
  messages: oldMessages.map((message) => new RemoveMessage({ id: message.id! })),
};
```

注意工具调用场景：

```text
AIMessage(tool_calls=[...])
ToolMessage(tool_call_id=...)
```

这两类消息通常必须成对保留。只删一半可能让模型提供商拒绝请求。

当前项目还没有工具调用时，可以先按简单 human / ai 对话处理。

## 9. 摘要：`summarizationMiddleware`

裁剪会丢信息。长任务更适合摘要：

```ts
import { createAgent, summarizationMiddleware } from "langchain";
import { MemorySaver } from "@langchain/langgraph";

const agent = createAgent({
  model: "openai:gpt-5-mini",
  tools: [],
  middleware: [
    summarizationMiddleware({
      model: "openai:gpt-5-mini",
      trigger: { tokens: 4000 },
      keep: { messages: 20 },
    }),
  ],
  checkpointer: new MemorySaver(),
});
```

摘要适合：

- 长时间任务规划。
- 长文档协作。
- 多轮 debug。
- 不希望早期关键信息被简单裁剪掉。

不建议第一版就做摘要。先用固定条数裁剪更容易验证。

## 10. 访问短期 state

工具里可以通过 `ToolRuntime` 读取 state：

```ts
const getCurrentUser = tool(
  async (_, runtime) => {
    return runtime.state.userId ?? "unknown";
  },
  {
    name: "get_current_user",
    description: "读取当前 state 中的 userId",
    schema: z.object({}),
  },
);
```

工具也可以返回 state update。更复杂时可以用 `Command` 更新 state 和 messages。

## 11. 当前项目的推荐阶段

当前 `mini-agent-langchain` 建议分四步：

```text
阶段 1：手写 Memory + 最近 20 条
阶段 2：session JSONL 落盘
阶段 3：createAgent + MemorySaver
阶段 4：middleware 管理 trim / summary
```

验收：

```text
> 我叫张三
> 我刚才说我叫什么？
```

再验证隔离：

```text
> /thread fresh
> 我刚才说我叫什么？
```
