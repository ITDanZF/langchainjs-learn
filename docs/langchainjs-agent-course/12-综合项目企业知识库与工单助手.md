# 12. 综合项目：企业知识库与工单助手

## 本章目标

前面章节分别学习了：

- ChatModel。
- Prompt 和 LCEL。
- Tools。
- ReAct Agent。
- LangGraph 状态图。
- 记忆与检查点。
- RAG。
- 多工具业务助手。
- 工程化、评估和观测。

本章把这些能力组合成一个综合项目：

```text
企业知识库与工单助手
```

它可以：

- 回答内部文档问题。
- 查询工单状态。
- 根据问题决定是否检索知识库。
- 根据问题决定是否调用业务工具。
- 在多轮会话中保留上下文。
- 输出可评估的结构化答案。

## 1. 应用能力设计

用户可能提出这些问题：

```text
请总结 docs 目录下的课程结构
```

```text
查询 open 状态的工单，并说明优先处理哪一个
```

```text
刚才你提到的课程结构里，哪几章适合加入 LangChain.js 示例？
```

这类问题不是单纯聊天，需要组合多种能力。

## 3. 工具边界

本项目需要四类工具：

| 工具 | 作用 |
| --- | --- |
| `list_files` | 查看目录结构 |
| `read_file` | 阅读具体文档 |
| `search_text` | 搜索关键词 |
| `query_ticket` | 查询业务工单 |

RAG 检索不一定作为普通 tool 暴露，也可以放在图节点里。

经验上：

- 确定性的业务操作适合做 tool。
- 检索和上下文拼装适合做 graph node。
- 高风险副作用操作要加审批节点。

## 4. Agent 行为规则

创建 `src/agents/knowledge-ticket-agent.ts`：

```ts
import { createReactAgent } from "@langchain/langgraph/prebuilt";
import { SystemMessage } from "@langchain/core/messages";
import { chatModel } from "../models/chat";
import { tools } from "../tools";

export const knowledgeTicketAgent = createReactAgent({
  llm: chatModel,
  tools,
  messageModifier: new SystemMessage([
    "你是企业知识库与工单助手。",
    "你可以阅读文档、搜索文本、查询工单。",
    "当用户询问项目、文档、课程或代码结构时，优先使用文件类工具获取事实。",
    "当用户询问工单、故障、客服或处理状态时，优先使用工单工具。",
    "回答必须说明依据来自文档、工具结果还是模型推断。",
    "如果信息不足，不要编造，明确说明缺少什么。",
  ].join("\n")),
});
```

不同版本的 LangChain.js 可能会调整预制 agent 的参数名。如果 `messageModifier` 不可用，请按当前版本文档改成等价配置。

## 5. CLI 入口

`src/index.ts`：

```ts
import { HumanMessage } from "@langchain/core/messages";
import { knowledgeTicketAgent } from "./agents/knowledge-ticket-agent";

const input = process.argv.slice(2).join(" ").trim();

if (!input) {
  console.error("请输入任务");
  process.exit(1);
}

const result = await knowledgeTicketAgent.invoke({
  messages: [new HumanMessage(input)],
});

console.log(result.messages.at(-1)?.content);
```

运行：

```bash
npm run dev "请阅读 docs 目录，总结目前有哪些教程"
```

## 6. 加入多轮上下文

如果要支持连续对话，可以把预制 agent 和 checkpointer 结合。

思路是：

```text
同一个用户 → 固定 thread_id
不同用户 → 不同 thread_id
```

调用时传入：

```ts
const config = {
  configurable: {
    thread_id: "demo-user-1",
  },
};

const result = await knowledgeTicketAgent.invoke(
  {
    messages: [new HumanMessage(input)],
  },
  config,
);
```

如果当前预制 agent 版本没有直接暴露 checkpointer 配置，就用第 06 章的 `StateGraph` 自定义图来承载记忆。

## 7. 加入评估用例

`src/evals/cases.ts`：

```ts
export const evalCases = [
  {
    input: "请阅读 docs 目录，总结课程结构",
    expectedKeywords: ["课程", "Node.js", "LangChain"],
  },
  {
    input: "查询 open 状态的工单",
    expectedKeywords: ["open", "T-1001"],
  },
  {
    input: "搜索 LangGraph 相关文档",
    expectedKeywords: ["LangGraph"],
  },
];
```

评估的重点不是要求每次输出完全一致，而是检查关键事实是否保留。

## 8. 项目升级方向

这个综合项目还可以继续升级：

- 把假工单换成真实 HTTP API。
- 给 `run_command` 加审批节点。
- 把 RAG 检索接入 LangGraph。
- 给不同用户保存不同会话。
- 接入 Web UI 或 TUI。
- 用 LangSmith 记录每次工具调用。
- 为高频问题增加缓存。
