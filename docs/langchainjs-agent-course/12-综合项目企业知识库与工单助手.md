# 12. 基础篇里程碑：企业知识库与任务助手

## 本章目标

本章把 01-11 章已经实现的能力整合成一次阶段性验收。

这不是最终企业架构，而是基础篇的第一个可用版本。完成后，你应该能连续运行：

```bash
mini-agent ask "解释这个项目"
mini-agent chat --thread team-a
mini-agent index docs
mini-agent run --graph "阅读 docs，整理 Agent CLI 学习路线并给出引用"
mini-agent eval
```

## 3. 典型企业任务

### 任务一：理解项目

```bash
mini-agent run --graph "阅读当前项目结构，说明各模块职责"
```

期望工具调用：

- `list_files`
- `read_file`
- `search_text`

期望输出：

- 模块职责。
- 依据文件。
- 后续改进建议。

### 任务二：文档问答

```bash
mini-agent index docs
mini-agent run --graph "哪些章节讲了工具调用和权限控制？请列出来源"
```

期望工具调用：

- `search_docs`
- 必要时 `read_file`

期望输出：

- 主题总结。
- 引用来源。
- 章节建议。

### 任务三：测试分析

```bash
mini-agent run --graph "运行测试并总结失败原因"
```

期望工具调用：

- `run_command` 执行 `npm test`
- 必要时读取失败文件

期望输出：

- 测试结果。
- 失败原因。
- 修复建议。

## 4. 企业级验收清单

功能验收：

- `ask` 能正常调用模型。
- `ask` 默认流式输出。
- `run` 能调用文件工具。
- `run --graph` 能完成工具循环。
- `chat` 能通过 `thread_id` 保留上下文。
- `index` 能创建本地文档索引。
- `eval` 能运行固定用例。

安全验收：

- 文件访问限制在 `AGENT_WORKSPACE`。
- 工具输出有长度限制。
- 高风险命令被拒绝或要求确认。
- 日志不污染标准输出。
- 错误信息对用户可理解。

工程验收：

- `npm run typecheck` 通过。
- `npm run test` 通过。
- `npm run build` 通过。
- 配置集中在 `src/config/env.ts`。
- CLI 不直接包含业务细节。

## 5. 从基础篇进入企业增强篇

到本章为止，`mini-agent-langchain` 已经完成基础闭环：

```text
CLI → Model → Tools → Agent → Graph → Memory → RAG → Eval
```

这条主干会被 13–22 章继续复用，而不是重写：

| 基础篇已有能力 | 企业增强篇如何复用 |
| --- | --- |
| `createChatModel()` | 第 13 章规划器、第 17 章模型路由继续复用 |
| `allTools` | 第 14 章审批、第 15 章编辑、第 18 章执行器继续复用 |
| `task-graph.ts` | 第 14 章加入 approval node，第 18 章扩展多 Agent 节点 |
| `search_docs` | 第 16 章升级为高级 RAG 和知识治理 |
| `run_command` | 第 14 章接入人工审批，第 15 章用于验证 patch |
| `eval` | 第 13–22 章持续扩展回归用例 |

## 6. 后续路线

当这个 CLI 稳定后，可以继续扩展：

| 方向 | 说明 |
| --- | --- |
| 持久化记忆 | 用 SQLite / Postgres 保存 thread |
| 审批流 | LangGraph 中加入 human-in-the-loop 节点 |
| 多 Agent | planner、researcher、executor、reviewer 分工 |
| 企业系统 | 接入工单、监控、CI、知识库 API |
| 权限体系 | 按用户、项目、环境控制工具能力 |
| Web 服务 | 把核心模块封装成 HTTP API |
| UI | 为 chat 和任务执行提供 Web 界面 |

## 7. 增强篇入口

建议下一步直接进入：
