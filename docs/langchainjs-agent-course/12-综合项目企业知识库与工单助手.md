# 12. 综合项目：企业知识库与任务助手

## 本章目标

本章把前面所有能力整合成 `mini-agent-langchain` 的完整形态。

最终命令：

```bash
mini-agent ask "解释这个项目"
mini-agent chat --thread team-a
mini-agent index docs
mini-agent run --graph "阅读 docs，整理 Agent CLI 学习路线并给出引用"
mini-agent eval
```

## 2. 最终能力矩阵

| 命令 | 能力 | 依赖模块 |
| --- | --- | --- |
| `ask` | 单轮问答 | `chains/ask.ts` |
| `ask --no-stream` | 非流式问答 | `chains/ask.ts` |
| `run` | 工具 Agent 执行 | `agents/task-agent.ts` |
| `run --graph` | 可控状态机执行 | `graph/task-graph.ts` |
| `chat` | 多轮交互会话 | `memory/thread.ts` + checkpointer |
| `index` | 本地文档索引 | `rag/index-docs.ts` |
| `eval` | 回归评估 | `evals/run-eval.ts` |

## 3. 典型企业任务

### 任务一：理解项目

```bash
mini-agent run --graph "阅读当前项目结构，说明各模块职责"
```

期望工具调用：

- `list_files`
- `read_file`
- `search_text`

期望输出：

- 模块职责。
- 依据文件。
- 后续改进建议。

### 任务二：文档问答

```bash
mini-agent index docs
mini-agent run --graph "哪些章节讲了工具调用和权限控制？请列出来源"
```

期望工具调用：

- `search_docs`
- 必要时 `read_file`

期望输出：

- 主题总结。
- 引用来源。
- 章节建议。

### 任务三：测试分析

```bash
mini-agent run --graph "运行测试并总结失败原因"
```

期望工具调用：

- `run_command` 执行 `npm test`
- 必要时读取失败文件

期望输出：

- 测试结果。
- 失败原因。
- 修复建议。

## 4. 企业级验收清单

功能验收：

- `ask` 能正常调用模型。
- `ask` 默认流式输出。
- `run` 能调用文件工具。
- `run --graph` 能完成工具循环。
- `chat` 能通过 `thread_id` 保留上下文。
- `index` 能创建本地文档索引。
- `eval` 能运行固定用例。

安全验收：

- 文件访问限制在 `AGENT_WORKSPACE`。
- 工具输出有长度限制。
- 高风险命令被拒绝或要求确认。
- 日志不污染标准输出。
- 错误信息对用户可理解。

工程验收：

- `npm run typecheck` 通过。
- `npm run test` 通过。
- `npm run build` 通过。
- 配置集中在 `src/config/env.ts`。
- CLI 不直接包含业务细节。

## 5. 后续路线

当这个 CLI 稳定后，可以继续扩展：

| 方向 | 说明 |
| --- | --- |
| 持久化记忆 | 用 SQLite / Postgres 保存 thread |
| 审批流 | LangGraph 中加入 human-in-the-loop 节点 |
| 多 Agent | planner、researcher、executor、reviewer 分工 |
| 企业系统 | 接入工单、监控、CI、知识库 API |
| 权限体系 | 按用户、项目、环境控制工具能力 |
| Web 服务 | 把核心模块封装成 HTTP API |
| UI | 为 chat 和任务执行提供 Web 界面 |
