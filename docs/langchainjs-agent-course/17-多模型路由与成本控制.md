# 17. 多模型路由与成本控制

## 本章目标

前面所有能力默认使用同一个模型。本章把模型调用层抽象成路由器，让 ask、plan、edit、rag 等不同任务选择不同模型。

企业 Agent 不应该所有任务都用同一个模型。简单任务用便宜模型，复杂推理用强模型，embedding 用专门模型。

本章新增：

```text
src/models/router.ts
src/models/profiles.ts
```

## 1. 为什么需要模型路由

不同任务成本差异很大：

| 任务 | 推荐模型类型 |
| --- | --- |
| CLI 简单问答 | 快速便宜模型 |
| 复杂规划 | 强推理模型 |
| 代码修改 | 代码能力强的模型 |
| RAG 答案 | 稳定长上下文模型 |
| embedding | 专用 embedding 模型 |

模型路由可以控制成本和质量。

## 2. 模型配置

`.env.example` 增加：

```bash
FAST_MODEL=deepseek-chat
REASONING_MODEL=deepseek-reasoner
CODE_MODEL=deepseek-chat
EMBEDDING_MODEL=text-embedding-3-small
```

创建 `src/models/profiles.ts`：

```ts
export type ModelProfile = "fast" | "reasoning" | "code" | "rag";

export const modelProfiles = {
  fast: { temperature: 0.2 },
  reasoning: { temperature: 0 },
  code: { temperature: 0 },
  rag: { temperature: 0.1 },
} satisfies Record<ModelProfile, { temperature: number }>;
```

## 3. 模型路由器

创建 `src/models/router.ts`：

```ts
import { ChatOpenAI } from "@langchain/openai";
import { env } from "../config/env.js";
import { modelProfiles, type ModelProfile } from "./profiles.js";

export function createModelFor(profile: ModelProfile) {
  const config = modelProfiles[profile];

  const model = profile === "reasoning"
    ? env.REASONING_MODEL
    : profile === "code"
      ? env.CODE_MODEL
      : env.FAST_MODEL;

  return new ChatOpenAI({
    model,
    temperature: config.temperature,
    apiKey: env.DEEPSEEK_API_KEY,
    configuration: { baseURL: env.DEEPSEEK_BASE_URL },
  });
}
```

## 4. 在模块中使用

- `ask` 使用 `fast`。
- `planner` 使用 `reasoning`。
- `patch-planner` 使用 `code`。
- `rag answer` 使用 `rag`。

这样不同能力可以独立调优。

## 5. 成本日志

每次模型调用记录：

```text
profile
model
duration_ms
input_tokens
output_tokens
estimated_cost
```

即使一开始拿不到 token，也应该记录模型、耗时和任务类型。

## 6. 降级策略

企业系统必须考虑模型失败：

```text
reasoning model 超时
  ↓
降级到 fast model
  ↓
提示用户结果可能不如预期
```

或者：

```text
主供应商失败 → 备用供应商
```

## 7. 验收

```bash
mini-agent plan "为 mini-agent-langchain 设计发布前检查流程"
```

规划任务应该走 reasoning profile，而普通 `ask` 走 fast profile。

## 8. 企业级思考

模型路由是成本控制的核心。一个企业 Agent 的费用失控，通常不是单次调用贵，而是所有小任务都用了大模型。
