# 00. 课程总览：从可对话 CLI 走向 Agent Runtime

## 当前起点

当前 `mini-agent-langchain` 已经不是空项目。它已经有了一个可以继续演进的 Agent CLI 雏形：

```text
src/
  main.ts                    # 启动 Bootstrap，并进入 CLI 循环
  bootstrap/index.ts          # 初始化目录、配置和默认 AgentRuntime
  cli/index.ts                # commander + readline 交互循环
  config/index.ts             # 读写 ~/.mini-agent/config.json
  workspace/index.ts          # 创建 ~/.mini-agent、logs、sessions、workspace
  model/Model.ts              # ChatOpenAI + PromptTemplate + stream/invoke
  model/index.ts              # AgentModel / AgentRuntime 管理雏形
  Memory/index.ts             # 记忆模块占位
```

这套结构已经能支撑下一步学习：让对话具备“记忆”。

## 本课程的主线

课程主线从现在开始不再是“先搭完整架构”，而是围绕一个持续演进的问题：

```text
一个能流式回复的 CLI，如何一步步变成真正的 Agent Runtime？
```

接下来的能力顺序是：

```text
单轮模型调用
  ↓
CLI 多轮输入
  ↓
短期记忆 messages
  ↓
thread_id 与会话隔离
  ↓
session 落盘
  ↓
LangGraph checkpointer
  ↓
工具调用
  ↓
可控 Agent Loop
  ↓
长期记忆、RAG、多 Agent
```

## 为什么现在应该学记忆

当前 `main.ts` 每次拿到用户输入后，会直接调用：

```ts
const result = await runTime.AgentRuntime.model.stream(input);
await PrintStream(result);
```

这意味着每一轮输入都是独立的。用户问：

```text
> 我叫张三
> 我叫什么？
```

如果没有记忆，第二轮模型无法稳定知道第一轮说过什么。

因此下一步不应该急着做复杂工具或多 Agent，而应该先补上最基本的上下文能力。

## 记忆的学习层次

### 第一层：进程内短期记忆

在 `src/Memory/index.ts` 中维护当前进程内的 messages：

```text
thread_id -> messages[]
```

这一步不需要 LangGraph，也不需要数据库。它的目标是理解消息历史如何进入模型上下文。

### 第二层：session 落盘

把每个 thread 的消息追加到：

```text
~/.mini-agent/sessions/<thread-id>.jsonl
```

这一步让 CLI 退出后仍然可以恢复会话。

### 第三层：LangGraph checkpointer

学习官方的短期记忆方式：

```ts
const checkpointer = new MemorySaver();
const graph = builder.compile({ checkpointer });

await graph.invoke(
  { messages: [{ role: "user", content: "hi! i am Bob" }] },
  { configurable: { thread_id: "1" } },
);
```

这一步把“自己维护 messages”升级为“图状态的线程级持久化”。

### 第四层：长期记忆 store

长期记忆不是当前对话历史，而是跨会话有价值的用户或应用信息：

```text
用户偏好
项目约定
常用命令
稳定事实
```

LangGraph 官方用 `store` 承载长期记忆，用 `checkpointer` 承载短期线程状态。

## 当前课程与官方概念的对应关系

| 当前项目概念 | 官方概念 | 说明 |
| --- | --- | --- |
| `CLI.CLILoop` | conversation loop | 接收用户输入的循环 |
| `Model.stream()` | chat model streaming | 流式模型输出 |
| `Memory/index.ts` | short-term memory 起点 | 当前将要实现的消息历史 |
| `~/.mini-agent/sessions` | session persistence | 本地会话落盘目录 |
| `threadId` | `configurable.thread_id` | 区分不同会话 |
| 未来 `StateGraph` | LangGraph graph state | 显式状态图 |
| 未来 `MemorySaver` | checkpointer | 线程级持久化 |
| 未来长期记忆文件/数据库 | store | 跨会话记忆 |

## 每章验收标准

每一章都要满足：

- 新能力能用当前项目命令跑起来。
- 概念能映射回实际代码文件。
- 不提前引入没有用上的复杂模块。
- 不把短期记忆、长期记忆、RAG 混在一起。
- 能说明这一步之后下一步为什么自然出现。
