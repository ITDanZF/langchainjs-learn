# 00. 进阶课程总览

## 本课程要解决什么问题

第一版课程让你理解 Agent 的底层构成：

```text
ChatMessage
LLMClient
Tool
ToolRegistry
AgentRuntime
Memory
Permission
```

第二版课程要进一步解决两个问题：

1. 如何用 LangChain.js 快速实现这些能力。
2. 如何用 LangGraph.js 构建更复杂、可控、可恢复的智能体应用。

## 为什么 LangChain.js 是重点

手写 Agent Runtime 有助于理解原理，但真实业务里会遇到更多复杂问题：

- 多模型适配。
- Prompt 组合。
- 工具参数校验。
- 工具调用链路。
- 流式输出。
- 多轮会话。
- RAG 检索。
- 工作流分支。
- 运行观测。
- 评估和回归测试。

LangChain.js 把这些能力封装成可组合模块。学会它，可以更快构建复杂应用。

## 和第一版课程的关系

| 第一版手写概念 | 第二版框架概念 |
| --- | --- |
| `ChatMessage` | `HumanMessage`、`AIMessage`、`SystemMessage` |
| `LLMClient` | `ChatModel` |
| `Tool` | `tool()` |
| `ToolRegistry` | tools array |
| `AgentRuntime` | `createReactAgent()` / `StateGraph` |
| `Memory` | checkpointer / `MemorySaver` |
| 工作流模式 | LCEL / LangGraph |
| 日志与调试 | LangSmith tracing |

第一版帮助你知道框架在做什么。第二版帮助你真正把框架用起来。

## 学习节奏

建议节奏：

- 第 1 天：完成 01 到 03，跑通模型、Prompt 和 LCEL。
- 第 2 天：完成 04 到 05，跑通工具调用 agent。
- 第 3 天：完成 06 到 07，掌握 LangGraph 状态和记忆。
- 第 4 天：完成 08，做一个 RAG 问答应用。
- 第 5 天：完成 09 到 11，做复杂业务助手并补工程化。

## 最终验收

最终你应该能输入：

```text
帮我阅读 docs 目录，总结课程结构，并指出哪些章节适合加入示例代码
```

Agent 应该能够：

1. 调用文件列表工具。
2. 读取相关 Markdown。
3. 根据任务决定是否搜索关键词。
4. 汇总结构化结果。
5. 在多轮会话中记住上下文。
6. 输出可追踪、可评估的答案。
