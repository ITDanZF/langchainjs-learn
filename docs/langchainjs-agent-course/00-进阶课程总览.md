# 00. 课程总览：从空项目到企业级 Agent CLI

## 本课程的主线

你已经创建了 `mini-agent-langchain` 文件夹。本课程会直接以它作为项目根目录，从零开始构建。

最终目标不是写一个临时 demo，而是形成一个企业项目骨架：

```text
mini-agent-langchain/
  package.json
  tsconfig.json
  .env.example
  src/
    cli.ts
    config/
    models/
    prompts/
    chains/
    tools/
    agents/
    graph/
    memory/
    rag/
    evals/
    observability/
    utils/
  tests/
  docs/
```

## 为什么从 CLI 开始

CLI 是学习 Agent 的最佳入口：

- 输入输出简单，容易调试。
- 能自然接入文件、搜索、命令等工具。
- 很适合模拟真实开发助手。
- 后续可以复用核心模块接入 Web、服务端 API 或企业 IM。

企业级 Agent 不等于一开始就复杂，而是每一步都保持边界清晰、可测试、可替换。

## 能力演进路线

### 阶段一：基础 CLI

产物：

```bash
mini-agent ask "你好"
```

你会学到：

- TypeScript 项目初始化。
- `commander` 子命令设计。
- 配置加载与校验。
- ChatModel 封装。
- Prompt 基础设计。

### 阶段二：工具 Agent

产物：

```bash
mini-agent run "阅读 README.md 并总结项目"
```

你会学到：

- LangChain.js `tool()`。
- `zod` 参数校验。
- 文件系统工具。
- `createAgent()` 工具调用循环。
- 工具结果和最终回答的边界。

### 阶段三：可控运行时

产物：

```bash
mini-agent run --graph "搜索 docs 中和 RAG 有关的章节"
```

你会学到：

- LangGraph 状态建模。
- 节点、边、条件路由。
- 中断、权限确认和错误恢复的设计位置。
- 为什么企业 Agent 需要显式状态机。

### 阶段四：记忆与知识库

产物：

```bash
mini-agent chat --thread project-a
mini-agent index docs
```

你会学到：

- `thread_id` 会话隔离。
- checkpointer 短期记忆。
- Markdown 文档加载、切分、索引和检索。
- 带引用的答案。

### 阶段五：企业化

产物：

```bash
mini-agent eval
npm run test
npm run build
```

你会学到：

- 日志与追踪。
- 错误码与用户友好提示。
- 回归评估。
- 成本、超时、重试、权限。
- npm bin 发布。

## 最终架构图

```text
CLI
 ├─ ask  ── AskChain ── ChatModel
 ├─ chat ── GraphAgent + Memory
 ├─ run  ── Agent Runtime ── Tools
 ├─ index ─ RAG Indexer
 └─ eval ── Evaluation Runner

Tools
 ├─ list_files
 ├─ read_file
 ├─ search_text
 ├─ search_docs
 └─ run_command

Enterprise Layer
 ├─ config validation
 ├─ logging / tracing
 ├─ permission policy
 ├─ tests / evals
 └─ deployment scripts
```

## 每章验收标准

每章结束都必须满足：

- 项目能运行。
- 新增能力有明确命令可验收。
- 代码放在清晰模块里。
- 不为了赶进度破坏安全边界。
