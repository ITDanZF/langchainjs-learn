# 00. 课程总览：从空项目到企业级 Agent CLI

## 本课程的主线

你已经创建了 `mini-agent-langchain` 文件夹。本课程会直接以它作为项目根目录，从零开始构建。

最终目标不是写一个临时 demo，而是形成一个企业项目骨架：

```text
mini-agent-langchain/
  package.json
  tsconfig.json
  .env.example
  src/
    cli.ts
    config/
    models/
    prompts/
    chains/
    tools/
    agents/
    graph/
    memory/
    rag/
    evals/
    observability/
    utils/
  tests/
  docs/
```

## 为什么从 CLI 开始

CLI 是学习 Agent 的最佳入口：

- 输入输出简单，容易调试。
- 能自然接入文件、搜索、命令等工具。
- 很适合模拟真实开发助手。
- 后续可以复用核心模块接入 Web、服务端 API 或企业 IM。

企业级 Agent 不等于一开始就复杂，而是每一步都保持边界清晰、可测试、可替换。

## 能力演进路线

### 阶段一：基础 CLI

产物：

```bash
mini-agent ask "你好"
```

你会学到：

- TypeScript 项目初始化。
- `commander` 子命令设计。
- 配置加载与校验。
- ChatModel 封装。
- Prompt 基础设计。

### 阶段二：工具 Agent

产物：

```bash
mini-agent run "阅读 README.md 并总结项目"
```

你会学到：

- LangChain.js `tool()`。
- `zod` 参数校验。
- 文件系统工具。
- `createAgent()` 工具调用循环。
- 工具结果和最终回答的边界。

### 阶段三：可控运行时

产物：

```bash
mini-agent run --graph "搜索 docs 中和 RAG 有关的章节"
```

你会学到：

- LangGraph 状态建模。
- 节点、边、条件路由。
- 中断、权限确认和错误恢复的设计位置。
- 为什么企业 Agent 需要显式状态机。

### 阶段四：记忆与知识库

产物：

```bash
mini-agent chat --thread project-a
mini-agent index docs
```

你会学到：

- `thread_id` 会话隔离。
- checkpointer 短期记忆。
- Markdown 文档加载、切分、索引和检索。
- 带引用的答案。

### 阶段五：企业化基础

产物：

```bash
mini-agent eval
npm run test
npm run build
```

你会学到：

- 日志与追踪。
- 错误码与用户友好提示。
- 回归评估。
- 成本、超时、重试、权限。
- npm bin 发布。

### 阶段六：复杂任务执行

产物：

```bash
mini-agent plan "分析项目发布风险"
mini-agent run --planned "检查项目是否可以发布"
```

你会学到：

- 任务规划器。
- 结构化执行计划。
- Planner / Executor / Reviewer 多 Agent 协作。
- 长任务的执行、复盘和重试。

### 阶段七：安全行动能力

产物：

```bash
mini-agent edit "给 CLI 添加一个 tools 命令"
mini-agent run --graph "安装依赖并运行测试"
```

你会学到：

- Human-in-the-loop 审批。
- 补丁式代码编辑。
- 高风险命令拦截。
- 工具权限、资源权限和审计日志。

### 阶段八：平台化能力

产物：

```bash
mini-agent server
mini-agent tools
```

你会学到：

- 服务化 API。
- 任务队列和事件流。
- 插件化工具生态。
- 多模型路由和成本控制。
- 高级 RAG 与知识库治理。

| --- | --- | --- |
| 规划器 | 13 | 复杂任务先规划再执行 |
| 审批流 | 14 | 高风险操作必须人工确认 |

| 高级 RAG | 16 | 增量索引、权限过滤、引用治理 |
| 多模型路由 | 17 | 控制成本并提升复杂任务质量 |
| 多 Agent | 18 | 规划、执行、审查分工 |
| 服务化 | 19 | 接入 Web、队列和企业 IM |
| 安全合规 | 20 | 权限、审计、Prompt 注入防护 |
| 插件生态 | 21 | 持续扩展业务工具 |
| 架构蓝图 | 22 | 从 CLI 走向企业 Agent 平台 |

## 章节依赖关系

课程不是 22 个独立主题，而是一条依赖链：

```text
01 项目骨架
  → 02–03 模型调用能力
  → 04 工具协议
  → 05 Agent 执行
  → 06–07 Graph 和记忆
  → 08–09 RAG 与命令工具
  → 10–12 工程闭环
  → 13–18 复杂任务、审批、编辑、多 Agent
  → 19–22 服务化、安全、插件、平台化
```

所以 13–22 不是“额外功能列表”，而是对 01–12 的企业级升级。

## 每章验收标准

每章结束都必须满足：

- 项目能运行。
- 新增能力有明确命令可验收。
- 代码放在清晰模块里。
- 不为了赶进度破坏安全边界。
