# 09. 多工具业务助手

## 本章目标

前面已经学过：

- ChatModel。
- Prompt。
- LCEL。
- Tool。
- ReAct Agent。
- LangGraph。
- Memory。
- RAG。

本章要把这些能力组合起来，做一个更接近真实业务的多工具助手。

目标能力：

```text
用户提出复杂任务
  ↓
Agent 判断需要哪些能力
  ↓
列目录 / 读文件 / 搜索文本 / 调业务 API
  ↓
整合工具结果
  ↓
输出结构化答案
```

## 1. 多工具助手的设计

一个复杂助手不应该只有一个工具。

例如“分析项目文档结构”可能需要：

- `list_files`：查看目录。
- `read_file`：读取关键文件。
- `search_text`：搜索关键词。
- `summarize`：总结内容。

真实业务里还可能需要：

- 查询订单。
- 查询客户。
- 创建工单。
- 调用内部 API。
- 生成报告。

## 2. 创建 searchTextTool

创建 `src/tools/search-text.ts`：

```ts
import { tool } from "@langchain/core/tools";
import { readFile, readdir } from "node:fs/promises";
import { join } from "node:path";
import { z } from "zod";

async function collectFiles(dir: string): Promise<string[]> {
  const entries = await readdir(dir, { withFileTypes: true });
  const files: string[] = [];

  for (const entry of entries) {
    const path = join(dir, entry.name);

    if (entry.isDirectory()) {
      files.push(...await collectFiles(path));
    } else {
      files.push(path);
    }
  }

  return files;
}

export const searchTextTool = tool(
  async ({ dir, keyword }) => {
    const files = await collectFiles(dir);
    const matches: string[] = [];

    for (const file of files) {
      const content = await readFile(file, "utf-8").catch(() => "");

      if (content.includes(keyword)) {
        matches.push(file);
      }
    }

    return matches.length > 0 ? matches.join("\n") : "未找到匹配文件";
  },
  {
    name: "search_text",
    description: "在指定目录中搜索包含关键词的文本文件。适合定位文档、代码或配置。",
    schema: z.object({
      dir: z.string().describe("要搜索的目录"),
      keyword: z.string().describe("要搜索的关键词"),
    }),
  },
);
```

这个示例为了教学保持简单。真实项目里可以用 `fast-glob` 和更严格的文件过滤。

## 3. 创建业务 API 工具

很多 agent 应用最终都会连接业务系统。

创建 `src/tools/business-api.ts`：

```ts
import { tool } from "@langchain/core/tools";
import { z } from "zod";

const fakeTickets = [
  { id: "T-1001", title: "登录失败", status: "open" },
  { id: "T-1002", title: "支付回调异常", status: "investigating" },
];

export const queryTicketTool = tool(
  async ({ status }) => {
    const tickets = fakeTickets.filter((ticket) => ticket.status === status);
    return JSON.stringify(tickets, null, 2);
  },
  {
    name: "query_ticket",
    description: "按状态查询工单列表。适合回答客服、运维和业务支持相关问题。",
    schema: z.object({
      status: z.string().describe("工单状态，例如 open 或 investigating"),
    }),
  },
);
```

先用假数据，是为了把重点放在 agent 和工具集成上。

## 5. 业务助手 Prompt

复杂应用需要更明确的系统规则。

创建 `src/agents/business-agent.ts`：

```ts
import { createReactAgent } from "@langchain/langgraph/prebuilt";
import { SystemMessage } from "@langchain/core/messages";
import { chatModel } from "../models/chat";
import { tools } from "../tools";

export const businessAgent = createReactAgent({
  llm: chatModel,
  tools,
  messageModifier: new SystemMessage([
    "你是一个企业内部智能助手。",
    "你可以读取文件、搜索文本、查询业务工单。",
    "在调用工具前先判断是否真的需要工具。",
    "回答时说明你使用了哪些信息来源。",
    "如果工具结果不足，请明确说明限制。",
  ].join("\n")),
});
```

如果你使用的 LangChain.js 版本里参数名不是 `messageModifier`，请查看当前版本文档。有些版本会使用 `stateModifier` 或其他配置方式。

## 6. 调用业务助手

```ts
import { HumanMessage } from "@langchain/core/messages";
import { businessAgent } from "./agents/business-agent";

const input = process.argv.slice(2).join(" ").trim();

if (!input) {
  console.error("请输入任务");
  process.exit(1);
}

const result = await businessAgent.invoke({
  messages: [new HumanMessage(input)],
});

console.log(result.messages.at(-1)?.content);
```

测试：

```bash
npm run dev "请查看 docs 目录，告诉我有哪些课程文档"
```

或者：

```bash
npm run dev "查询 open 状态的工单，并总结当前待处理事项"
```

## 7. 多工具应用的风险

工具越多，能力越强，风险也越高：

- 工具描述不清导致误调用。
- 参数 schema 太宽导致输入不可控。
- 文件读取可能泄露敏感信息。
- 命令执行可能破坏环境。
- 业务 API 可能造成真实副作用。

所以复杂应用必须配合权限控制和观测。
