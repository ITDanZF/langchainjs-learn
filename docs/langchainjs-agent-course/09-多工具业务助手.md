# 09. 企业工具箱与权限：加入受限命令执行

## 本章目标

本章让 Agent 能执行低风险命令，并建立最小权限策略。

完成后可以运行：

```bash
npm run dev -- run --graph "运行 npm test，并总结结果"
```

## 1. 命令工具的价值与风险

命令执行工具能让 Agent 做很多开发任务：

- 运行测试。
- 搜索代码。
- 查看 git 状态。
- 执行构建。

但它也可能造成严重破坏：

- 删除文件。
- 重置代码。
- 下载并执行未知脚本。
- 泄露环境变量。

所以企业 Agent 的命令工具必须默认保守。

## 2. 命令策略

创建 `src/tools/command-policy.ts`：

```ts
const allowedCommands = new Set(["pwd", "ls", "cat", "rg", "grep", "npm", "node", "npx", "git"]);
const blockedTokens = ["rm", "sudo", "curl", "wget", "--hard", "reset", "clean", "rebase"];

export function validateCommand(command: string, args: string[]) {
  if (!allowedCommands.has(command)) {
    throw new Error(`Command is not allowed: ${command}`);
  }

  const text = [command, ...args].join(" ");
  if (blockedTokens.some((token) => text.includes(token))) {
    throw new Error(`Command contains blocked token: ${text}`);
  }
}

export function requiresApproval(command: string, args: string[]) {
  if (command === "git") return true;
  if (command === "npm" && ["install", "add", "publish"].some((arg) => args.includes(arg))) return true;
  return false;
}
```

这是教学版策略。企业环境中建议用配置文件维护白名单，并按用户角色授权。

## 3. run_command 工具

创建 `src/tools/run-command.ts`：

```ts
import { tool } from "@langchain/core/tools";
import { spawn } from "node:child_process";
import { z } from "zod";
import { workspaceRoot } from "../utils/workspace.js";
import { requiresApproval, validateCommand } from "./command-policy.js";

export const runCommandTool = tool(
  async ({ command, args }) => {
    validateCommand(command, args);

    if (requiresApproval(command, args)) {
      return `APPROVAL_REQUIRED: ${command} ${args.join(" ")}`;
    }

    return new Promise<string>((resolve) => {
      const child = spawn(command, args, {
        cwd: workspaceRoot,
        shell: false,
      });

      let output = "";
      child.stdout.on("data", (chunk) => (output += chunk.toString()));
      child.stderr.on("data", (chunk) => (output += chunk.toString()));
      child.on("close", (code) => {
        resolve(`exitCode=${code}\n${output.slice(0, 12000)}`);
      });
    });
  },
  {
    name: "run_command",
    description: "在工作区执行低风险命令，例如 pwd、ls、rg、npm test。高风险命令会被拒绝或要求确认。",
    schema: z.object({
      command: z.string().describe("命令名，不包含参数"),
      args: z.array(z.string()).default([]).describe("命令参数数组"),
    }),
  },
);
```

## 4. 加入工具集合

修改 `src/tools/index.ts`：

```ts
import { runCommandTool } from "./run-command.js";

export const commandTools = [runCommandTool];
export const allTools = [
  ...filesystemTools,
  searchDocsTool,
  ...commandTools,
];
```

## 5. 更完整的审批流

本章的 `APPROVAL_REQUIRED` 只是一个信号。企业级实现应该放在 LangGraph 中：

```text
agent 节点
  ↓
发现高风险工具调用
  ↓
approval 节点询问用户
  ↓
用户确认后执行 tools 节点
```

这样权限是运行时控制，而不是混在工具内部。

## 6. Prompt 增加命令规则

在 `taskAgentPrompt` 中补充：

```text
使用 run_command 时：
1. 优先选择只读命令。
2. 不要尝试删除、重置、下载执行脚本。
3. 如果工具返回 APPROVAL_REQUIRED，向用户说明需要确认，不要假装已经执行。
```
