# 新闻监控技能（News Monitor）

用于追踪、解析并量化新闻对全球预测市场与金融市场的影响，自动将事件与相关市场匹配，输出可执行的交易洞察与实时警报。

## 触发场景

在以下情境中使用本技能：

- 用户要求监控影响预测市场或股票市场的新闻源。
- 需要分析某条新闻对特定市场（A股、港股、美股、预测市场等）价格的影响。
- 需要为股票或预测市场构建新闻驱动的交易信号。
- 需要量化市场情绪并生成实时警报。
- 需要撰写与金融市场、新闻分析相关的SEO优化内容。

## 核心能力

- **新闻采集与过滤**：从RSS、API、社交媒体、交易所公告等渠道抓取新闻，按关键词、来源、地区、时间窗口过滤。
- **自动市场匹配**：基于实体识别（NER）与主题建模，将新闻事件映射到相关市场或个股。
- **价格影响分析**：计算新闻发布后市场的价格变动、波动率变化与成交量异动。
- **市场情绪量化**：使用情感分析模型对新闻进行打分，生成市场情绪指数。
- **实时新闻警报**：根据预定义阈值触发警报，输出事件摘要、影响市场、预期价格方向与置信度。
- **SEO优化输出**：生成包含关键词的标题、摘要与元描述，提升内容可发现性。

## 标准工作流程

1. **定义监控范围**：明确目标市场（A股、港股、美股、预测市场等）、标的列表、关键词、时间窗口与数据源。
2. **采集新闻**：调用新闻API、交易所公告、RSS源或社交媒体接口，获取原始新闻数据。
3. **清洗与去重**：去除重复、低质量或与目标无关的内容，处理多语言文本。
4. **提取关键信息**：识别时间、地点、实体、事件类型、涉及市场与情感倾向。
5. **匹配受影响市场**：将新闻与相关市场、板块或个股关联，给出匹配理由。
6. **分析价格影响**：对比新闻发布前后的价格、成交量与波动率数据。
7. **生成洞察与警报**：输出市场移动洞察、交易信号与实时警报。
8. **SEO优化（如需要）**：为报告或文章添加关键词、标题与元描述。

## 新闻-市场自动匹配方法

- **实体匹配**：提取新闻中的组织、人物、地点、政策、行业等实体，与市场标的或描述对比。
- **关键词匹配**：使用预定义关键词库（如公司名称、行业术语、经济指标、政策名称）进行相似度计算。
- **主题建模**：通过LDA/BERTopic识别新闻主题，映射到对应的市场类别或板块。
- **地区映射**：根据新闻涉及的国家、地区或交易所，优先匹配对应市场（如A股、港股、美股、欧股）。
- **时间窗口**：优先匹配新闻发布时间前后成交量或价格显著变化的市场。

## 价格影响分析框架

1. **事件窗口定义**：确定新闻发布前后时间段（如[-30min, +60min]），考虑时区差异。
2. **价格变动计算**：
   - 绝对变动：`ΔP = P_post - P_pre`
   - 相对变动：`Return = (P_post - P_pre) / P_pre × 100%`
3. **波动率变化**：计算事件窗口内的标准差或与历史波动率的比值。
4. **成交量异动**：对比事件窗口成交量与近期平均成交量。
5. **方向判断**：结合情感分析与价格走势，判断市场可能的移动方向。
6. **置信度评分**：基于数据质量、样本量、事件重要性与模型输出给出0-100分。

## 市场情绪量化

- **情感极性**：正面、负面、中性。
- **情感强度**：0-1之间的强度分数。
- **情绪指数**：综合多新闻情感得分，生成加权市场情绪指数。
- **极化度**：衡量市场观点分歧程度，辅助判断波动性。
- **跨区域情绪对比**：对比不同市场（如A股vs美股）对同一事件的情绪反应差异。

## 数据源与地区覆盖

根据目标市场选择合适的数据源：

- **A股/港股**：
  - 交易所公告（上交所、深交所、港交所）
  - 财经新闻（财新、华尔街见闻、东方财富、同花顺）
  - 政策与监管动态（央行、证监会、国务院政策）
- **美股/国际**：
  - 交易所公告（SEC filings、NYSE、NASDAQ）
  - 财经新闻（Bloomberg、Reuters、CNBC、Financial Times）
  - 宏观数据（美联储、欧央行、非农数据、CPI）
- **预测市场**：
  - 平台数据（Polymarket、Kalshi、PredictIt等）
  - 社交媒体与事件追踪源

## 多语言与时区处理

- **语言支持**：同时处理中文、英文及其他主要语言新闻，必要时进行翻译或统一编码。
- **时区对齐**：将所有时间统一转换为目标市场本地时间或UTC，避免跨市场分析偏差。
- **交易时间过滤**：区分不同市场的交易时段，判断新闻是否在交易时间内发布。

## SEO优化指南

为生成的报告或文章优化搜索引擎可见性：

- **标题**：包含核心关键词，控制在60字符以内。示例：`A股新闻监控：政策利好对新能源板块的价格影响分析`
- **关键词**：新闻监控、预测市场、A股、港股、美股、价格关联、市场情绪、量化分析、新闻警报。
- **元描述**：150字符以内，概括内容价值。示例：`实时监控影响A股、港股与美股的新闻事件，自动匹配受影响市场并提供价格影响分析与市场情绪量化。`
- **正文结构**：使用H2/H3标题、列表与表格，提升可读性。
- **内部链接**：引用相关市场分析或历史报告。

## 注意事项

- 新闻影响具有滞后性和不确定性，避免将单一新闻作为唯一交易依据。
- 区分相关性与因果性，结合多数据源交叉验证。
- 不同市场受监管、交易机制、投资者结构影响，同一新闻在不同市场可能产生不同效果。
- 实时数据依赖可靠API，注意频率限制、数据延迟与时区差异。
- 情感分析模型可能存在偏见，必要时人工复核关键事件。
